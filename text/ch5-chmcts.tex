\chapter{\label{ch:5-chmcts}Convex Hull Monte Carlo Tree Search}

    \minitoc

    This chapter considers multi-objective Monte Carlo Tree Search 
    (MOMCTS) for the decision support scenario (\todo{ref back to ch2 fig}) with linear utilities, where the agent must plan for an unknown weight vector $\bff{w}$ over the objectives. Two approaches are considered in this chapter, the first following the same design paradigm as UCT where an algorithm for CMAB problems (Section \ref{sec:2-1-3-contextual-mab}) is applied at each decision node; the second is to maintain a convex hull value set at each node in the 
    search tree, representing the set of achievable expected returns 
    across all possible weightings over the objectives which is updated using convex hull backups (Section \ref{sec:2-6-1-chvi}). 
    
    This chapter is organised as follows:
    \begin{itemize}
        \item Section \ref{sec:5-1-intro} motivates the 
            multi-objective decision making framework used in this 
            chapter, and generalises the notions of simple and cumulative regret to 
            \textit{contextual simple regret} and 
            \textit{contextual cumulative regret} for the multi-objective setting;
        \item Section \ref{sec:5-2-czt} introduces 
            \textit{Contextual Zooming for Trees} (CZT), a 
            MOMCTS algorithm following the same design paradigm 
            as UCT;
        \item Section \ref{sec:5-3-chmcts} generalises prior work 
            in MOMCTS to stochastic environments through 
            \textit{convex hull backups};
        \item Section \ref{sec:5-5-results} presents empirical 
            results on Deep Sea Treasure environments and a range 
            of multi-objective environments from the 
            Multi-Objective Gymnasium library \todo{cite?};
        \item Section \ref{sec:5-6-theory} provides a brief 
            theoretical discussion of the MOMCTS algorithms 
            introduced in this chapter.
    \end{itemize}

\section{Introduction and Motivation}
\label{sec:5-1-intro}

    This section motivates the use of \textit{contextual tree search} 
    for planning in multi-objective settings, and highlights why 
    consistent action selection with respect to a fixed objective 
    is a necessary component of effective MOMCTS. The notions of 
    cumulative and simple regret are then generalised to the 
    multi-objective setting.

\subsection{Consistent Action Selection}

    \begin{figure}
        \centering
        \includegraphics[width=0.6\textwidth]{figures/todo.jpg}
        \caption[A motivating example MOMDP for contextual tree search.]{Tree 
        MOMDP, with two actions at the root. From the first action, rewards of 
        $(1,0)$, $(0.5,0.5)$ and $(0,0.75)$ are available. From the second, 
        rewards of $(0.75,0)$, $(0.5,0.5)$ and $(0,1)$ are available. The 
        convex hull value set is formed of $\{(1,0),(0,1)\}$.}
        \label{fig:5:motivating_example}
    \end{figure}

    Consider the MOMDP illustrated in Figure \ref{fig:5:motivating_example}, 
    with two objectives. The convex hull value set of the MOMDP is $\{(1,0),(0,1)\}$: the reward $(1,0)$ is only reachable via the left action, and $(0,1)$ only via the right action. The optimal action at the root node therefore depends entirely on the weight vector $\bff{w}$: if $w_1 > w_2$ then the left action is optimal, and if $w_2 > w_1$ then the right action is optimal.

    Once the agent has taken the left action from the root node, it has implicitly committed to optimising towards the first objective (i.e. $w_1 > w_2$) because the rewards available from the right action are no longer reachable. Although all actions at the immediate child nodes are \textit{locally optimal} (i.e. they lead to a value in the local convex hull value set), only the action leading to $(1,0)$ is consistent with the implicit commitment made at the root node.

        \centering
        \includegraphics[width=0.6\textwidth]{figures/todo.jpg}
        \caption[A motivating example MOMDP for contextual tree search.]{\todo{update caption with }}
        \label{fig:5:motivating_vanmoffaert}
    \end{figure}

    The problem of \textit{consistent action selection} is also discussed by \todo{citet vanmoffaert14a}. They compute and track Pareto-optimal policies using an adaptation of Q-learning and highlight that even if an action is \textit{locally optimal}, it may not be \textit{globally optimal}. To demonstrate this, they use an example with three optimal paths (Figure \ref{fig:5:motivating_vanmoffaert}), where the black path alternates between multiple optimal paths, taking only locally optimal actions at each step but fails to follow any single globally optimal path.

\subsection{Contextual Tree Search}
\label{sec:5-1-1-contextual_tree_search}

    \todo{citet vanmoffaert and double check} solve the consistent action selection problem by explicitly tracking the optimal policies across the full MOMDP. The approach taken in this thesis is to consider a \textit{contextual tree search}, where a context weight vector $\bff{w}$ is sampled at the start of each trial and used to guide action selection throughout that trial. In the context of MORL with linear utilities, this is sufficient to resolve the consistent action selection problem, ensuring that all actions within a trial are consistent with a single objective.

    In the \thtspp\ewe framework, the context weight $\bff{w}$ is sampled during the \samplecontext\ewe function (Section \ref{sec:2-4-1-thts}) and passed to all subsequent steps of the trial, including the search policy and backup functions.

    \begin{defn}
        \label{def:5:context_policy}
        A \textnormal{contextual policy} $\pi(a|s;\bff{w})$ is a 
        policy that may depend on the context weight vector 
        $\bff{w}\in\Delta^{D}$, where $\pi(a|s;\bff{w})$ gives 
        the probability of selecting action $a$ at state $s$ under 
        weight $\bff{w}$.

        A policy $\pi$ is \textnormal{context independent} if 
        $\pi(a|s;\bff{w}) = \pi(a|s;\bff{w}')$ for all 
        $\bff{w},\bff{w}'\in\Delta^{D}$ and all $s\in\cl{S}$, 
        $a\in\cl{A}$. Otherwise, $\pi$ is 
        \textnormal{context dependent}.
    \end{defn}

    To reason about the performance of a contextual tree search algorithm, the notions of cumulative and simple regret are generalised to \textit{contextual cumulative regret} and \textit{contextual simple regret}. These definitions reduce to the standard definitions (Definitions \ref{def:2:cumulative_regret} and \ref{def:2:simple_regret}) when $D=1$.

    \begin{defn}
        \label{def:5:contextual_cumulative_regret}
        The \textnormal{contextual cumulative regret} after $n$ 
        rounds is:
        \begin{align}
            \text{CR}_n = \sum_{m=1}^n \left( 
                \max_{a\in\cl{A}} (\bff{w}^m)^\top \bff{Q}^*(s_0, a) 
                - (\bff{w}^m)^\top \bff{Q}^*(s_0, a^m) 
            \right),
        \end{align}
        where $a^m$ is the action selected in round $m$ and 
        $\bff{Q}^*(s,a)$ is the optimal multi-objective Q-value 
        vector at $(s,a)$.
    \end{defn}

    \begin{defn}
        \label{def:5:contextual_simple_regret}
        The \textnormal{contextual simple regret} of a 
        recommendation policy $\psi$ at state $s_0$ is:
        \begin{align}
            \text{SR}(\bff{w}, \psi) = \max_{a\in\cl{A}} 
                \bff{w}^\top \bff{Q}^*(s_0,a) 
                - \bff{w}^\top \bff{Q}^*(s_0, \psi(s_0;\bff{w})),
        \end{align}
        where $\psi(s_0;\bff{w})$ is the action recommended by 
        $\psi$ under weight $\bff{w}$.
    \end{defn}

    Several prior MOMCTS works \todo{cite}, discussed in Section \todo{ref ch3}, use a context-independent search policy. It is shown in Section \ref{sec:5-6-theory} that any context-independent search policy suffers linear contextual cumulative regret, and any context-independent recommendation policy suffers linear contextual simple regret.

\section{Contextual Zooming for Trees}
\label{sec:5-2-czt}

    \begin{figure}
        \centering
        \includegraphics[width=0.5\textwidth]{figures/todo.jpg}
        \caption[Design paradigm for CZT.]{The design paradigm 
        for CZT. UCB is lifted to UCT by applying it at each node 
        of the search tree. CZ generalises UCB to contextual bandits.

    A common design paradigm in MCTS is to adapt a MAB algorithm to run at each decision node of the search tree, treating the node as a (non-stationary) bandit problem. UCT follows this paradigm by running UCB at each node (Section \ref{sec:2-4-2-uct}). 
    
    The \textit{Contextual Zooming} (CZ) algorithm \cite{cz,cz_long} is a generalisation of the UCB algorithm to contextual bandits (Section \ref{sec:2-1-3-contextual-mab}). Following the same methodology, \textit{Contextual Zooming for Trees} (CZT) runs the CZ algorithm at each decision node of the search tree. The relationship between these algorithms is illustrated in Figure \ref{fig:5:czt_diagram}.
    
    CZT is presented is in this section in the context of MOMCTS, where the context is the weight vector $\bff{w}$ sampled at the start of each trial. Although CZ, and by extension CZT, is not motivated by the exploration setting this thesis focusses on, it is \bd{introduced to fill a gap in the MOMCTS literature}.

\subsection{The Contextual Zooming Algorithm}
\label{sec:5-2-1-cz}

    The Contextual Zooming algorithm \cite{cz,cz_long} solves contextual multi-armed bandit problems (Section \ref{sec:2-1-3-contextual-mab}) where the similarity space $\cl{W}\times\cl{X}$ is compact and the mean reward function $\mu$ is Lipschitz continuous. 
    
    CZ is presented \bd{below} in the context of contextual tree search (Section \ref{sec:5-1-1-contextual_tree_search}), where the context is the weight vector $\bff{w}$ sampled at the start of each trial (i.e. $\cl{W} = \Delta^D$).

    The CZ algorithm maintains a set of \textit{active balls} that cover the similarity space. Let $A^m(s)$ denote the set of active balls at decision node $s$ with $m=N(s)$ visits, where a ball with centre $(w,x)$ and radius $r$ is defined as $\{(w',x')\in\cl{W}\times\cl{X} \mid d((w,x),(w',x')) < r\}$.

    \bd{Each ball will have an associated visit count} $M(B^m(s))$ \bd{and average reward} $\bar{y}^m(B^m(s))$. \todo{update notation to be a V value function}.

    Each round, CZ applies two rules: a \textit{selection rule}, which selects a ball $B^m(s)$ from $A^m(s)$, as part of the \selectaction\ewe function; and an \textit{activation rule}, which optionally adds a ball with smaller radius to $A^m(s)$, as part of the \backupv\ewe function.
    
\paragraph{Selection Rule}

    Given the context weight $w^n\sim\samplecontext(s)$ for the $n$th trial, the selection rule proceeds as follows. Let $M(B^m(s))$ be the number of times ball $B^m(s)\in A^m(s)$ has been selected in the first $m$ rounds, and $\bar{y}^m(B)$ the 
    average reward received when ball $B$ was selected:
    \begin{align}
        N^m(B) &= \sum_{i=1}^m \one[B=B^i], 
            \label{eq:5:cz_num_ball_selected} \\
        \bar{y}^m(B) &= \frac{1}{N^m(B)} \sum_{i=1}^m 
            y^i \one[B=B^i]. 
            \label{eq:5:cz_ball_avg_reward}
    \end{align}

    Let $r(B)$ denote the radius of ball $B$ and let 
    $\dom^m(B)$ be the domain of ball $B$ on the $m$th round, 
    which excludes all balls of smaller radius:
    \begin{align}
        \dom^m(B) = B - \left( \bigcup_{B'\in A^m:r(B')<r(B)} 
            B' \right). \label{eq:5:cz_domain}
    \end{align}

    The \textit{confidence radius} $\conf^m(B)$ of ball $B$ on 
    the $m$th round is:
    \begin{align}
        \conf^m(B) = 4\sqrt{\frac{\log T}{1+N^{m-1}(B)}}. 
            \label{eq:5:cz_conf_radius}
    \end{align}

    Let $\relevant^m(w,A^m)$ denote the set of balls relevant 
    for context $w$, i.e. balls containing an arm $x$ such that 
    $(w,x)$ is in the domain of the ball:
    \begin{align}
        \relevant^m(w,A^m) = \{ B\in A^m \mid \exists x\in\cl{X}.\ 
            (w,x)\in\dom^m(B)\}.
    \end{align}

    The ball $B^m$ selected on the $m$th round with context 
    $w^m$ is chosen as:
    \begin{align}
        I^m(B) &= \bar{y}^{m-1}(B) + r(B) + \conf^m(B), 
            \label{eq:5:cz_pre_index} \\
        B^m &= \argmax_{B\in\relevant^m(w^m,A^m)} r(B) + 
            \min_{B'\in A^m} \left( I^m(B') + d(B,B') \right), 
            \label{eq:5:cz_ball_selection}
    \end{align}
    where $d(B,B')$ is the distance between the centres of balls 
    $B$ and $B'$ according to the metric $d$. The arm sampled 
    by CZ is then drawn uniformly from the domain of $B^m$: 
    $x^m_{\cz} \sim \text{Uniform}(\{x\in\cl{X} \mid 
    (w^m,x)\in\dom^m(B^m)\})$.

    The activation rule is applied when the confidence radius 
    of the selected ball $B^m$ falls below its radius, i.e. 
    if $\conf^{m+1}(B^m) \leq r(B^m) < \conf^m(B^m)$. In this 
    case, a new ball $B'$ is added with radius 
    $r(B')=\frac{1}{2}r(B^m)$ and centre $(w^m,x^m)$, so that 
    $A^{m+1}=A^m\cup\{B'\}$. Otherwise $A^{m+1}=A^m$.

\citet{cz} introduce the Contextual Zooming (CZ) algorithm to solve CMAB problems. CZ achieves a contextual regret of $O(n^{\frac{1+c}{2+c}}\log(T))$ over the first $n$ rounds, using the \textit{doubling trick} \cite{doubling_trick}, where $c$ is the \textit{(Lebesgue) covering dimension} of the similarity space $\cl{W}\times\cl{X}$. Within this thesis, the covering dimension coincides with the standard notion of dimension: $\cl{W}\times\cl{X}$ has covering dimension $c$, if it embeds into $\bb{R}^{c}$.

The CZ algorithm uses a set of balls in the similarity space, called \textit{active balls}. Let $A^m$ denote the set of active balls at the start of the $m$th round. A ball with centre $(w,x)$ and radius $r$ is defined as $\{(w',x')\in\cl{W}\times\cl{X}\ |\ d((w,x),(w',x')) < r \}$. 

Each round, CZ applies two rules: a \textit{selection rule}, which selects a ball $B^m$ from $A^m$; and an \textit{activation rule} which optionally adds a ball with smaller radius to $A^m$. 

Given a context $w^m$, the selection rule proceeds as follows: let $N^m(B)$ be the number of times that a ball $B\in A^m$ has been selected in the first $m$ rounds, and $\bar{y}^m(B)$ the average reward received over the first $m$ rounds when ball $B$ was selected.
\begin{align}
    N^m(B) &= \sum_{i=1}^m \one[B=B^m], \label{eq:2:cz_num_ball_selected} \\
    \bar{y}^m(B) &= \frac{1}{N^m(B)} \sum_{i=1}^m y^m \one[B=B^m]. \label{eq:2:cz_ball_avg_reward}
\end{align}

Let $r(B)$ denote the radius of ball $B$ and let $\dom^m(B)$ be the domain of ball $B$ on the $m$th round, which is the subset of $B$ that excludes all balls of smaller radius:
\begin{align}
    \dom^m(B) = B - \left( \bigcup_{B'\in A^m:r(B')<r(B)} B' \right). \label{eq:2:cz_domain}
\end{align}

The \textit{confidence radius} $\conf^m(B)$ of ball $B$ on the $m$th round is:
\begin{align}
    \conf^m(B) = 4\sqrt{\frac{\log T}{1+N^{m-1}(B)}}. \label{eq:2:cz_conf_radius}
\end{align}

Let $\relevant^m(w,A^m)$ denote the set of balls that are relevant for context $w$ on the $m$th round, which is the set of balls that contain an arm $x$ such that $(w,x)$ is in the domain of the ball:
\begin{align}
    \relevant^m(w,A^m) = \{ B\in A^m | \exists x\in\cl{X}. (w,x)\in\dom^m(B)\}.
\end{align}

The ball $B^m$ selected on the $m$th round with context $w^m$ is chosen as follows:
\begin{align}
    I^m(B) &= \bar{y}^{m-1}(B) + r(B) + \conf^m(B), \label{eq:2:cz_pre_index} \\
    B^m &= \argmax_{B\in\relevant^m(w^m,A^m)} r(B) + \min_{B'\in A^m} \left( I^m(B') + d(B,B') \right), \label{eq:2:cz_ball_selection}
\end{align}
where $d(B,B')$ is the distance between the centres of balls $B,B'$ according to the metric $d$. 

Once a ball has been selected, the arm sampled by CZ, $x^m_{\cz}$, is sampled randomly from the domain of ball $B^m$ (from the set $\{x\in\cl{X}|(w^m,x)\in\dom^m(B^m)\}$).

The activation rule is applied when the confidence radius of the selected ball $B^m$ is smaller than its radius, i.e. if $\conf^{m+1}(B^m) \leq r(B^m) < \conf^m(B^m)$. In this case, a new ball $B'$ is added to the set of active balls, with radius $r(B')=\frac{1}{2}r(B^m)$ and centre $(w^m,x^m)$. If a new ball is added, then $A^{m+1}=A^m\cup \{B'\}$ and otherwise $A^{m+1}=A^m$.

This section introduces \textit{Convex Hull Monte Carlo Tree Search} (CHMCTS), which generalises the prior work in multi-objective MCTS \todo{ref ch3} to handle stochastic environments by using backups over \textit{Convex Hull Value Sets} from Chapter 2 \todo{ref ch2}. First the backups, which will be common across all of the following algorithms.

Let ALG be a substitute acronym for any of the following algorithms. At each decision node $s$ a value set is maintained $\Valg(s)$, and at chance nodes $(s,a)$ the value set $\Qalg(s)$ is maintained. The search policy, $\pialg(\cdot|s,\bff{w})$, is \textit{contextual} and will depend on a randomly sampled $\bff{w}\in\Delta^D$ for each trial. \bd{Assume for now that the search policy is defined, as it will differ for each algorithm.}

Given a context objective $\bff{w}\in\Delta^D$ and a trajectory $\tau=(s_0,a_0,\bff{r}_0,...,s_{h-1},a_{h-1},\bff{r}_{h-1},s_h)\sim\pialg$ \todo{want the w in the pialg bit somehow? Maybe make it superscript? But dont want it to mess with notation for theory.} the value estimates are updated for $t=h-1,...,0$:
\begin{align}
    \Valg(s_t) &\leftarrow \cprune \left( \bigcup_{a_t\in\cl{A}} \Qalg(s_t,a_t) \right), 
        \label{eq:5:chmcts_v_backup} \\
    \Qalg(s_t,a_t) &\leftarrow \bff{R}(s_t,a_t) + \sum_{s'\in\suc{s_t}{a_t}} \frac{N(s')}{N(s_t,a_t)}\Valg(s').

where the $\cprune$ operation is defined in \eqref{eq:2:cprune_def}, \bd{but repeated here for clarity} over an arbitrary set of vectors $\bfcl{V}$:
\begin{align}
    \cprune(\bfcl{V}) = \{\bff{v}\in\bfcl{V} | \exists \bff{w}\in\Delta^D. \forall \bff{v}'\in\bfcl{V}-\{\bff{v}\}. \bff{w}^\top \bff{v} > \bff{w}^\top \bff{v}' \}. \label{eq:5:cprune_def}
\end{align}

\end{align}

    A natural way to extend the UCT search policy for contextual tree search is to use the contextual value estimates \eqref{eq:5:contextual_value} and \eqref{eq:5:contextual_qvalue} in the UCT search policy. Call this algorithm \textit{UCT with Convex Hulls} (CH-UCT). This gives the following search policy:
    \begin{align}
        \pichuct(s|\bff{w}) &= \argmax_{a\in\cl{A}} \left[
            \Qchuctctx{\bff{w}}(s,a) + \cchuct \sqrt{\frac{\log(N(s))}{N(s,a)}} \right].
    \end{align}

    \bd{One issue is that the confidence interval terms are shared between all possible context objectives, and may assign a high confident, or tight bound, for contexts that haven't been explored down this path much.} A hueristic way to solve this is to a vectorised visit counts $\bff{M}(s)$, $\bff{M}(s,a)$. These values are backed up after a trial for context $\bff{w}$ as follows:
    \begin{align}
        \bff{M}(s) \leftarrow \bff{M}(s) + \bff{w}, \\
        \bff{M}(s,a) \leftarrow \bff{M}(s,a) + \bff{w}.
    \end{align}

    Then a contextual visit count can be defined, and used in the search policy as follows:
    \begin{align}
        M(s;\bff{w}) &= \bff{w}^\top \bff{M}(s) \\
        M(s,a;\bff{w}) &= \bff{w}^\top \bff{M}(s,a) \\
        \pichuct(s|\bff{w}) &= \argmax_{a\in\cl{A}} \left[
            \Qchuctctx{\bff{w}}(s,a) + \cchuct \sqrt{\frac{\log(M(s;\bff{w}))}{M(s,a;\bff{w})}} \right].
    \end{align}

    Note that the original value of $N(s)$ is still needed in the convex hull backups for Q-value sets, to empirically estimate the transition distribution $p(s'|s,a)\approx \frac{N(s')}{N(s,a)}$, however this can be recovered from the vectorised visit count by summing over the values of the vector:
    \begin{align}
        N(s) &= \sum_{i=1}^D M(s)_i \\
        N(s,a) &= \sum_{i=1}^D M(s,a)_i.
    \end{align}

% Chebyshev scalarisation proof
% Assumes standard amsmath / amssymb / amsthm setup.
 
\begin{prop}
    Let $u(w, x, z) = \min_{i} w_i \lvert x_i - z_i \rvert$ denote the Chebyshev
    scalarisation, and consider the vectors
    \[
      a = (10, 0), \quad b = c = (1, 1), \quad d = (0, 10),
    \]
    together with a reference point $z \in \mathbb{R}^2$ satisfying $z_i < \min\{a_i, b_i, c_i, d_i\}$
    componentwise. Let $w = (w_1, w_2)$ be any weight vector with $w_1, w_2 > 0$ and
    $w_1 + w_2 = 1$. Then
    \[
      u(w, b, z) > u(w, a, z) \quad \text{or} \quad u(w, c, z) > u(w, d, z).
    \]
\end{prop}
     
    \begin{proof}
    The componentwise inequality $z_i < \min\{a_i, b_i, c_i, d_i\}$ permits the absolute
    values to be dropped, so that
    \[
      u(w, x, z) = \min_{i} w_i (x_i - z_i)
    \]
    for each of $x \in \{a, b, c, d\}$. The shifted coordinates
    \[
      \alpha_i = a_i - z_i, \qquad
      \beta_i  = b_i - z_i = c_i - z_i, \qquad
      \delta_i = d_i - z_i
    \]
    are therefore all strictly positive. From $z_1 < 0$ and $z_2 < 0$ (which follow from
    $z < d$ and $z < a$ respectively), the following strict inequalities are obtained:
    \begin{equation}\label{eq:key-ineqs}
      \alpha_2 = -z_2 < 1 - z_2 = \beta_2,
      \qquad
      \delta_1 = -z_1 < 1 - z_1 = \beta_1.
    \end{equation}
    Since $b = c$, the equality $u(w, b, z) = u(w, c, z)$ holds throughout.
     
    Suppose, for contradiction, that both disjuncts fail, i.e.
    \begin{align}
      \min(w_1 \alpha_1,\, w_2 \alpha_2) &\geq \min(w_1 \beta_1,\, w_2 \beta_2), \label{eq:A} \\
      \min(w_1 \delta_1,\, w_2 \delta_2) &\geq \min(w_1 \beta_1,\, w_2 \beta_2). \label{eq:D}
    \end{align}
     
    From \eqref{eq:A}, the left-hand side is bounded above by $w_2 \alpha_2$, so
    \[
      w_2 \alpha_2 \;\geq\; \min(w_1 \beta_1,\, w_2 \beta_2).
    \]
    By \eqref{eq:key-ineqs} we have $w_2 \alpha_2 < w_2 \beta_2$, whence the minimum on
    the right cannot equal $w_2 \beta_2$. It follows that
    \begin{equation}\label{eq:from-A}
      \min(w_1 \beta_1,\, w_2 \beta_2) = w_1 \beta_1, \qquad \text{so} \qquad w_1 \beta_1 \leq w_2 \beta_2.
    \end{equation}
     
    By the analogous argument applied to \eqref{eq:D}, using $w_1 \delta_1 < w_1 \beta_1$,
    \begin{equation}\label{eq:from-D}
      \min(w_1 \beta_1,\, w_2 \beta_2) = w_2 \beta_2, \qquad \text{so} \qquad w_2 \beta_2 \leq w_1 \beta_1.
    \end{equation}
     
    Combining \eqref{eq:from-A} and \eqref{eq:from-D} yields $w_1 \beta_1 = w_2 \beta_2$.
    Substituting back into \eqref{eq:A} gives $w_2 \alpha_2 \geq w_1 \beta_1 = w_2 \beta_2$,
    i.e.\ $\alpha_2 \geq \beta_2$, contradicting \eqref{eq:key-ineqs}.
     
    The assumption that both disjuncts fail is therefore untenable, and at least one of
    $u(w, b, z) > u(w, a, z)$ or $u(w, c, z) > u(w, d, z)$ must hold.
    \end{proof}
     
    \begin{remark}
    The argument relies only on the strict inequalities $a_2 < b_2$ and $d_1 < c_1$
    (together with $b = c$). Consequently the proposition generalises: whenever a common
    point $b = c$ strictly dominates $a$ in one coordinate and $d$ in the other, no
    single Chebyshev weight vector can simultaneously render both $a$ preferable to $b$
    and $d$ preferable to $c$.
    \end{remark}
