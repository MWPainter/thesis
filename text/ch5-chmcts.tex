\chapter{\label{ch:5-chmcts}Convex Hull Monte Carlo Tree Search}

    \minitoc

    This chapter considers multi-objective Monte Carlo Tree Search 
    (MOMCTS) for the decision support scenario (Figure \ref{fig:2:mo_decision_support}) with linear utilities, where the agent must plan for an unknown weight vector $\bff{w}$ over the objectives. Primarily, this chapter aims to address \stochq.
    
    Two approaches are considered in this chapter, the first following the same design paradigm as UCT where an algorithm for CMAB problems (Section \ref{sec:2-1-3-contextual-mab}) is applied at each decision node; the second is to maintain a convex hull value set at each node in the 
    search tree, representing the set of achievable expected returns 
    across all possible weightings over the objectives which is updated using convex hull backups (Section \ref{sec:2-6-1-chvi}). 
    
    This chapter is organised as follows:
    \begin{itemize}
        \item Section \ref{sec:5-1-intro} motivates the 
            multi-objective decision making framework used in this 
            chapter, and generalises the notions of simple and cumulative regret to 
            \textit{contextual simple regret} and 
            \textit{contextual cumulative regret} for the multi-objective setting;
        \item Section \ref{sec:5-2-czt} introduces 
            \textit{Contextual Zooming for Trees} (CZT), a 
            MOMCTS algorithm following the same design paradigm 
            as UCT;
        \item Section \ref{sec:5-3-chmcts} generalises prior work 
            in MOMCTS to stochastic environments through 
            \textit{convex hull backups};
        \item Section \ref{sec:5-5-results} presents empirical 
            results on Deep Sea Treasure environments and a range 
            of multi-objective environments from the 
            Multi-Objective Gymnasium library \cite{mo_gym};
        \item Section \ref{sec:5-6-theory} provides a brief 
            theoretical discussion of the MOMCTS algorithms 
            introduced in this chapter.
    \end{itemize}

\section{Introduction and Motivation}
\label{sec:5-1-intro}

    This section motivates the use of \textit{contextual tree search} 
    for planning in multi-objective settings, and highlights why 
    consistent action selection with respect to a fixed objective 
    is a necessary component of effective MOMCTS. The notions of 
    cumulative and simple regret are then generalised to the 
    multi-objective setting.

\subsection{Consistent Action Selection}

    \begin{figure}
        \centering
        \includegraphics[width=0.6\textwidth]{figures/ch5/motivate_contextual_momdp.pdf}
        \caption[A motivating example MOMDP for contextual tree search.]{A motivating example MOMDP for contextual tree search. The convex hull value set at the root consists of $\{(4,0),(0,4)\}$, following the red paths. The optimal values at the root node require specific actions at the child nodes (blue), while at the child nodes all actions are \textit{locally optimal}.}
        \label{fig:5:motivating_example}
    \end{figure}

    Consider the MOMDP illustrated in Figure \ref{fig:5:motivating_example}, with two objectives. The convex hull value set of the MOMDP is $\{(4,0),(0,4)\}$: the reward $(4,0)$ is only reachable via the left action, and $(0,4)$ only via the right action. The optimal action at the root node therefore depends entirely on the weight vector $\bff{w}$: if $w_1 > w_2$ then the left action is optimal, and if $w_2 > w_1$ then the right action is optimal.

    Once the agent has taken the left action from the root node, it has implicitly committed to optimising towards the first objective (i.e. $w_1 > w_2$) because the rewards available from the right action are no longer reachable. Although all actions at the immediate child nodes are \textit{locally optimal} (i.e. they lead to a value in the local convex hull value set), only the action leading to $(4,0)$ is consistent with the implicit commitment made at the root node.

    \begin{figure}
        \centering
        \begin{subfigure}{0.45\textwidth}
            \centering
            \includegraphics[width=\textwidth]{figures/ch5/locally_optimal.pdf}
            \label{fig:5:motivating_vanmoffaert:sub:a}
        \end{subfigure}
        \begin{subfigure}{0.45\textwidth}
            \centering
            \includegraphics[width=\textwidth]{figures/ch5/locally_optimal_interleved.pdf}
            \label{fig:5:motivating_vanmoffaert:sub:b}
        \end{subfigure}
        \caption[Example of locally optimal actions that are not globally optimal.]{Example of locally optimal actions that are not globally optimal (reproduced from \citet{locally_optimal}). (a) The three globally optimal paths in green, red and blue. (b) The black path interleaves actions from the green, red and blue paths where actions are locally optimal at each step, but is a globally suboptimal path.}
        \label{fig:5:motivating_vanmoffaert}
    \end{figure}

    The problem of \textit{consistent action selection} is also discussed by \citet{locally_optimal}. They compute and track Pareto-optimal policies using an adaptation of Q-learning and highlight that even if an action is \textit{locally optimal}, it may not be \textit{globally optimal}. To demonstrate this, they use an example with three optimal paths (Figure \ref{fig:5:motivating_vanmoffaert}), where the black path alternates between multiple optimal paths, taking only locally optimal actions at each step but fails to follow any single globally optimal path.


\subsection{Contextual Tree Search}
\label{sec:5-1-1-contextual_tree_search}

    \citet{locally_optimal} solve the consistent action selection problem by explicitly tracking the optimal policies across the full MOMDP. The approach taken in this thesis is to consider a \textit{contextual tree search}, where a context weight vector $\bff{w}$ is sampled at the start of each trial and used to guide action selection throughout that trial. In the context of MORL with linear utilities, this is sufficient to resolve the consistent action selection problem, ensuring that all actions within a trial are consistent with a single objective.

    In the \thtspp\ewe framework, the context weight $\bff{w}$ is sampled during the \samplecontext\ewe function (Section \ref{sec:2-4-2-thts}) and passed to all subsequent steps of the trial, including the search policy and backup functions.

    \begin{defn}
        \label{def:5:context_policy}
        A \textnormal{contextual policy} $\pi(a|s;\bff{w})$ is a 
        policy that may depend on the context weight vector 
        $\bff{w}\in\Delta^{D}$, where $\pi(a|s;\bff{w})$ gives 
        the probability of selecting action $a$ at state $s$ under 
        weight $\bff{w}$.

        A policy $\pi$ is \textnormal{context independent} if 
        $\pi(a|s;\bff{w}) = \pi(a|s;\bff{w}')$ for all 
        $\bff{w},\bff{w}'\in\Delta^{D}$ and all $s\in\cl{S}$, 
        $a\in\cl{A}$. Otherwise, $\pi$ is 
        \textnormal{context dependent}.
    \end{defn}

    To reason about the performance of a contextual tree search algorithm, the notion simple regret is generalised to \textit{contextual simple regret}.

    % \begin{defn}
    %     \label{def:5:contextual_cumulative_regret}

    \begin{defn}
        \label{def:5:contextual_simple_regret}
        The \textnormal{contextual simple regret} of a recommendation policy $\psi$ at state $s\in\cl{S}$ is:
        \begin{align}
            \ctxsreg(s,\psi) = \bb{E}_{\bff{w}\sim U(\Delta^D)}\left[
                \max_{\pi'} {\bff{w}}^\top \bff{V}^{\pi'}(s) 
                - {\bff{w}}^\top \bff{V}^{\psi(\cdot;\bff{w})}(s)
            \right],
        \end{align}
        where $\bff{w}\sim U(\Delta^D)$ denotes a weight vector sampled uniformly from the unit $(D-1)$-dimensional simplex.
    \end{defn}

\section{Contextual Zooming for Trees}
\label{sec:5-2-czt}

    A common design paradigm in MCTS is to adapt a MAB algorithm to run at each decision node of the search tree, treating the node as a (non-stationary) bandit problem. UCT follows this paradigm by running UCB at each node (Section \ref{sec:2-4-2-uct}). 
    
    The \textit{Contextual Zooming} (CZ) algorithm \cite{cz,cz_long} is a generalisation of the UCB algorithm to contextual bandits (Section \ref{sec:2-1-3-contextual-mab}). Following the same methodology, \textit{Contextual Zooming for Trees} (CZT) runs the CZ algorithm at each decision node of the search tree. 
    
    CZT is presented in this section in the context of MOMCTS, where the context is the weight vector $\bff{w}$ sampled at the start of each trial. Although CZ, and by extension CZT, is not motivated by the exploration setting this thesis focuses on, it is introduced as a natural contextual analogue of UCT, and as a contextual search policy baseline for MOMCTS, where no such baseline currently exists.

\subsection{Algorithm Definition}
\label{sec:5-2-1-czt}

    Contextual Zooming operates over a similarity space (defined in Section \ref{sec:2-1-3-contextual-mab}), which in the case of CZT is the weight-action space $\Delta^D\times\cl{A}$. The following metric is defined over the weight-action space for use in CZT:
    %
    \begin{align}
        \dczt((\bff{w},a),(\bff{w}',a')) = \| \bff{w} - \bff{w}' \|_2 + C_{\textnormal{czt}}\one\left[a\neq a'\right], \label{eq:5:czt_metric}
    \end{align}
    %
    where $\Cczt>\sqrt{2}$ is a large constant such that the distance between any two actions is always greater than the distance between any two weights.

    CZ maintains a set of \textit{active balls} that cover the weight-action space. 
    In CZT, let $A(s,a)$ denote the set of active balls for action $a$ at decision node $s$, where a ball with centre $(\bff{w},a)$ and radius $r$ is defined as $\ball((\bff{w},a),r) = \{(\bff{w}',a')\in\Delta^D\times\cl{A} \mid d((\bff{w},a),(\bff{w}',a')) < r\}$. Let $A(s)$ be the set of all active balls at decision node $s$, i.e. $A(s) = \cup_{a\in\cl{A}} A(s,a)$.

    Each set of active balls is initialised to a single ball, using the centroid of the simplex $\frac{1}{D}\bff{1}$, where $\bff{1}$ is the vector of all ones. Specifically, 
    \begin{align}
        A^0(s,a) = \left\{ \ball\left(\left(\frac{1}{D}\bff{1},a\right),\sqrt{\frac{D-1}{D}}\right) \right\}. \label{eq:5:czt_initial_ball}
    \end{align}

    Each ball $B\in A(s,a)$ has an associated visit count $M(B)$ and average return value estimate $\Qczt(B)$. The visit count and value estimates are initialised to $M(B)=0$ and $\Qczt(B)=\bff{0}$ respectively.

    Two rules are applied to run contextual zooming: 
    \begin{itemize}
        \item the \textit{selection rule}, to select a ball (and action) from the set of active balls at the current decision node, as part of the \selectaction\ewe function;
        \item the \textit{activation rule}, which optionally adds a ball with smaller radius to the set of active balls, as part of the \backupv\ewe function.
    \end{itemize}
    
\paragraph{Selection Rule}

    Let $\bff{w}$ be the context weight sampled for the $n$th trial, let $r(B)$ denote the radius of a ball $B\in A(s,a)$. The \textit{domain} of $B$ in CZ consists of the portion of the ball not covered by any smaller active ball. Specifically,
    \begin{align}
        \dom(B) = B - \left( \bigcup_{B'\in A(s,a):r(B')<r(B)} 
            B' \right). \label{eq:5:czt_domain}
    \end{align}

    The \textit{confidence radius} $\conf(B)$ of ball $B$ is:
    \begin{align}
        \conf(B) = \bczt \sqrt{\frac{\log(N(s))}{1+M(B)}},
            \label{eq:5:czt_conf_radius}
    \end{align}
    where $\bczt>0$ is a UCB style bias parameter.

    Let $\relevant(s,a;\bff{w})$ denote the set of \textit{relevant} balls at a decision node $s$ for action $a$. A ball is relevant if $(\bff{w},a)$ is in the domain of the ball. Specifically,
    \begin{align}
        \relevant(s,a;\bff{w}) = \{ B \in A(s,a) \mid (\bff{w},a)\in\dom(B)\}, \\
        \relevant(s;\bff{w}) = \cup_{a\in\cl{A}} \relevant(s,a;\bff{w}).
    \end{align}

    Contextual Zooming defines a confidence interval term, called the \textit{pre-index}, around each ball, similar to UCB's confidence interval term:
    \begin{align}
        \Ipreczt(B;\bff{w}) = \bff{w}^\top \Qczt(B) + r(B) + \conf(B). \label{eq:5:cz_pre_index}
    \end{align}

    The full \textit{index} of a ball $B$ allows a newly activated ball with few visits to inherit an optimistic value estimate from nearby well-explored balls, ensuring it remains eligible for selection. Specifically,
    \begin{align}
        \Iczt(B;\bff{w}) = 
            r(B) + \min_{B'\in A(s)} \left( \Ipreczt(B';\bff{w}) + d(B,B') \right), \label{eq:5:cz_index}
    \end{align}

    where $d(B,B')$ is the distance between the centres of balls $B$ and $B'$ according to the metric $d$. In CZT the constant $\Cczt$ is set to a large enough value such that balls with a different action will never minimise the expression, so it suffices to consider only balls with the same action as $B$ in Equation (\ref{eq:5:cz_index}).

    To select a ball, the \selectball\ewe function chooses the ball from the set of relevant balls with the highest index:
    \begin{align}
        \selectball(s,\bff{w}) = \argmax_{B\in\relevant(s;\bff{w})} \Iczt(B).
    \end{align}

    If $\selectball(s,\bff{w}) = \ball((\bff{w}_B,a_B),r_B)$, then the CZT selection policy returns $\piczt(s;\bff{w})=a_B$.

\paragraph{Backups and Activation Rule}

    Let $\bff{\tau}_{:h}\sim\piczt$ be the trajectory sampled in the selection phase of CZT. Let $B_t$ be the ball selected at the decision node $s_t$. 

    The backup functions update the ball selection counts and the multi-objective average returns for $t=h-1,\ldots,0$:
    \begin{align}
        M(B_t) &\leftarrow M(B_t) + 1, \\
        \Qczt(B_t) &\leftarrow \frac{M(B_t)-1}{M(B_t)} \Qczt(B_t) + \frac{\Vinitbff(s_h) + \sum_{i=t}^{h-1} \bff{r}_i}{M(B_t)},
    \end{align}
    where $\Vinitbff(s_h)$ is the (multi-objective) heuristic value function.

    The \textit{activation rule} introduces a new ball with smaller radius when the following conditions are met:
    \begin{align}
        M(B_t) > \Mmin \quad \text{and} \quad \conf(B_t) \leq r(B_t),
    \end{align}
    %
    where $\Mmin$ is a minimum selection threshold, and when both conditions are met, a new ball $B_{\textnormal{new}} = \ball((\bff{w},a_t),\frac{r(B_t)}{2})$ is added to the set of active balls $A(s,a)$.

\paragraph{Recommendation Policy}

    To recommend an action in CZT, the ball with maximum linear utility from the set of active balls is selected. Specifically,
    \begin{align}
        \psiczt(s;\bff{w}) = \argmax_{a\in\cl{A}} \max_{B\in \relevant(s,a;\bff{w})} \bff{w}^\top \Qczt(B).
    \end{align}


\subsection{Differences from Contextual Zooming}
\label{sec:5-2-2-cz-diffs}

    Section \ref{sec:5-2-1-czt} described the CZT algorithm. The main adaptations from CZ are summarised below:
    \begin{enumerate}
        \item 
            In CZT, the set of active balls is specialised to a set for each action. In CZ, a single set of active balls is used across the entire \textit{similarity space} (defined in Section \ref{sec:2-1-3-contextual-mab}).
        \item \label{item:5:local1}
            CZ assumes that the algorithm is run for a known number of rounds $T$, and uses this value instead of the current number of visits at the decision node $N(s)$ in the confidence radii (Equation \ref{eq:5:czt_conf_radius}).
        \item 
            The activation rule of CZ does not have a hard lower bound on the number of ball visits required to add a new child ball. CZT makes this adaptation to account for item \ref{item:5:local1}, which results in smaller confidence radii.
        \item 
            In CZT, all balls contain exactly one action, whereas in CZ with a general continuous space of arms, balls cover many arms. After a ball is selected in CZ, an arm is sampled uniformly randomly from the domain of the selected ball. 
    \end{enumerate}

    CZ \cite{cz,cz_long} extends its regret analysis from a fixed number of rounds $T$ to an unknown number of rounds using the \textit{doubling trick} \cite{doubling_trick}. The doubling trick runs an independent instance of the algorithm for $T$ rounds, then $2T$ rounds, then $4T$ rounds and so on. In practice, however, discarding all accumulated data at each doubling interval is undesirable. Instead, in CZT the number of visits $N(s)$ at the current node is used directly in place of $T$, which avoids the need for the doubling trick at the cost of losing regret guarantees.

\section{Convex Hull Monte Carlo Tree Search}
\label{sec:5-3-chmcts}

    This section introduces \textit{Convex Hull Monte Carlo Tree Search} (CHMCTS), which generalises prior work in MOMCTS \ref{sec:3-5-momcts} to stochastic environments by maintaining \textit{convex hull value sets} at each node in the search tree and updating them using convex hull backups (Section \ref{sec:2-6-1-chvi}). When the MOMDP is deterministic and a search policy from prior work is used, CHMCTS reduces to the prior MOMCTS algorithms discussed in Chapter \ref{ch:3-relatedwork}.

    The common \thtspp\ewe framework and backup equations shared by all CHMCTS algorithms are given first. The search policies for each algorithm are then defined. Two practical optimisations are discussed in Section \ref{sec:5-3-6-optimisations}, and the section concludes with a brief discussion on the relationship between entropy and convex hull value sets.

\subsection{Common Framework}
\label{sec:5-3-1-framework}

    Let $\alg$ denote any of the CHMCTS algorithms defined in this section. At each decision node $s$, a convex hull value set $\Valg(s)$ is maintained, and at each chance node $(s,a)$, a convex hull Q-value set $\Qalg(s,a)$ is maintained. The specific form of the search policy varies between algorithms and is defined in the subsections below.

    Given a context weight $\bff{w}\in\Delta^D$ and a trajectory $\tau_{:h}=(s_0,a_0,\bff{r}_0,\ldots,s_{h-1},a_{h-1},\bff{r}_{h-1},s_h)\sim\pialg(\cdot|\cdot;\bff{w})$, the value set estimates are updated for $t=h-1,\ldots,0$:
    \begin{align}
        \Valg(s_t) &\leftarrow \cprune \left( 
            \bigcup_{a\in\cl{A}} \Qalg(s_t,a) \right), 
            \label{eq:5:chmcts_v_backup} \\
        \Qalg(s_t,a_t) &\leftarrow \bff{R}(s_t,a_t) + 
            \sum_{s'\in\suc{s_t}{a_t}} 
            \frac{N(s')}{N(s_t,a_t)}\Valg(s'). 
            \label{eq:5:chmcts_q_backup}
    \end{align}

    The $\cprune$ operation, defined in Equation 
    (\ref{eq:2:cprune_def}) and repeated here for convenience, 
    retains only those vectors in a set $\bfcl{V}$ that are 
    optimal for at least one weight vector:
    \begin{align}
        \cprune(\bfcl{V}) = \{\bff{v}\in\bfcl{V} \mid 
            \exists \bff{w}\in\Delta^D.\ 
            \forall \bff{v}'\in\bfcl{V}-\{\bff{v}\}.\ 
            \bff{w}^\top \bff{v} > \bff{w}^\top \bff{v}' \}. 
            \label{eq:5:cprune_def}
    \end{align}

    Since the context weight is passed to policies, it is useful to define \textit{contextual value estimates}, which extract a scalar value from a convex hull value set for a given weight $\bff{w}\in\Delta^D$:
    \begin{align}
        \Valgctx{\bff{w}}(s) &= \max_{\bff{v}\in\Valg(s)} 
            \bff{w}^\top \bff{v}, 
            \label{eq:5:contextual_value} \\
        \Qalgctx{\bff{w}}(s,a) &= \max_{\bff{q}\in\Qalg(s,a)} 
            \bff{w}^\top \bff{q}. 
            \label{eq:5:contextual_qvalue}
    \end{align}

    The recommendation policy selects the action with the highest contextual Q-value under the given weight:
    \begin{align}
        \psialg(s;\bff{w}) = \argmax_{a\in\cl{A}} 
            \Qalgctx{\bff{w}}(s,a). \label{eq:5:chmcts_rec}
    \end{align}

\subsection{Convex Hull UCT}
\label{sec:5-3-2-chuct}

    Another natural extension of UCT to the contextual multi-objective setting is to replace the scalar Q-value estimates (used in Section \ref{sec:2-4-2-uct}) with contextual value estimates from Equations (\ref{eq:5:contextual_value}) and (\ref{eq:5:contextual_qvalue}). The resulting algorithm, \textit{Convex Hull UCT} (CH-UCT), uses the following search policy:
    \begin{align}
        \pichuct(s;\bff{w}) = \argmax_{a\in\cl{A}} \left[
            \Qchuctctx{\bff{w}}(s,a) + \cchuct 
            \sqrt{\frac{\log(N(s))}{N(s,a)}} \right],
            \label{eq:5:chuct_policy}
    \end{align}
    where $\cchuct > 0$ is a bias parameter controlling exploration. The search policy is context-dependent, since the contextual Q-value $\Qchuctctx{\bff{w}}(s,a)$ depends on $\bff{w}$, ensuring that action selection is consistent with the sampled objective throughout each trial.

\subsection{Convex Hull BTS}
\label{sec:5-3-3-chbts}

    \textit{Convex Hull BTS} (CH-BTS) uses the BTS search policy (Section \ref{sec:4-2-1-bts}) within the CHMCTS framework, using contextual value estimates in place of the scalar Bellman value estimates:
    \begin{align}
        \rhochbts(a|s;\bff{w}) &\propto \exp\left(
            \frac{\Qchbtsctx{\bff{w}}(s,a)}
            {\alphachbts(N(s))}\right), 
            \label{eq:5:chbts_rho} \\
        \pichbts(a|s;\bff{w}) &= (1-\lambdachbts)
            \rhochbts(a|s;\bff{w}) 
            + \frac{\lambdachbts}{|\cl{A}|}, 
            \label{eq:5:chbts_policy} \\
        \lambda(s,\epsilon) &= \min\left(1, 
            \frac{\epsilon}{\log(e+N(s))}\right),
    \end{align}
    where $\alphachbts(x) > 0$ is a temperature function and $\epschbts \in [0,\infty)$ is an exploration parameter. As in BTS, the temperature function controls the trade-off between exploration and exploitation, and the exploration parameter ensures that all actions are eventually explored.

\subsection{Convex Hull CZT}
\label{sec:5-3-4-chczt}

    In single-objective MCTS, Bellman backups often outperform average return backups in the exploration setting (Chapter \ref{ch:4-dents}), since they propagate value information more directly through the tree by bootstrapping estimates from successor states rather than averaging over full trajectories. Convex hull 
    backups can be viewed as the multi-objective analogue of 
    Bellman backups: rather than maintaining a scalar average 
    return at each node, they maintain a set of achievable 
    value vectors that more accurately represents the range 
    of outcomes reachable from that node. The CZT ball value 
    estimates $\Qczt(B)$ provide a coarser representation, 
    averaging over a region of weight space rather than 
    capturing the full structure of achievable values.

    Since the CHMCTS backups (Equations 
    (\ref{eq:5:chmcts_v_backup}) and 
    (\ref{eq:5:chmcts_q_backup})) are independent of the 
    search policy, they can be combined with any search 
    policy, including CZT. The resulting algorithm, 
    \textit{Convex Hull CZT} (CH-CZT), runs the CZT 
    algorithm but maintains additional convex hull value 
    sets at each node and uses the recommendation policy 
    from Equation (\ref{eq:5:chmcts_rec}) in place of the 
    CZT recommendation policy. This combines the contextual 
    exploration of CZT with the richer value representation 
    of CHMCTS.

\subsection{Prior Search Policies}
\label{sec:5-3-5-prior}

    The MOMCTS algorithms discussed in Chapter \ref{ch:3-relatedwork} can be recovered within the CHMCTS framework by substituting appropriate search policies. In general, prior work uses search policies of the following form:
    \begin{align}
        \pialg(a|s;\bff{w}) = \argmax_{a\in\cl{A}} \left[
            \zetaalg(\Qalg(s,a);\bff{w}) + \calg 
            \sqrt{\frac{\log(N(s))}{N(s,a)}} \right],
            \label{eq:5:prior_policy}
    \end{align}
    where $\zetaalg$ is a scalarisation function that maps a 
    convex hull Q-value set to a scalar, and $\calg > 0$ is 
    a bias parameter. Notably, when $\zetaalg$ does not depend 
    on $\bff{w}$, the search policy is context-independent, 
    which as discussed in Section \ref{sec:5-1-intro} leads to 
    linear contextual cumulative regret.
 

\paragraph{Hypervolume CHMCTS (H-CHMCTS).}

    \citet{hv_momcts_1} use the \textit{hypervolume indicator} (Definition \ref{def:2:hypervolume}) as the scalarisation function $\zetaalg$ in the search policy. That is, $\zetahchmcts(\bfcl{V};\bff{w}) = \HV(\bfcl{V};\rref)$, where $\rref$ is reference point for the hypervolume indicator and is a parameter of the algorithm.
    
\paragraph{Chebyshev CHMCTS (C-CHMCTS).}

    \citet{chebyshev_momcts} use the \textit{Chebyshev utility}:

    \begin{defn}
        \label{def:5:chebyshev_utility}
        Let $\bff{v},\ruto\in\bb{R}^D$ and $\bff{w}\in\Delta^D$. The \textnormal{Chebyshev utility} of $\bff{v}$ with respect to $\bff{w}$ and $\ruto$ is:
        \begin{align}
            \ucheby(\bff{v};\bff{w},\ruto) = - \max_{i=1,\ldots,D} w_i |v_i - \rutocomp_i|.
        \end{align}
    \end{defn}

    The scalarisation function for C-CHMCTS is:
    \begin{align}
        \zetcchmcts(\bfcl{V};\bff{w},\ruto) = \max_{\bff{v}\in\bfcl{V}} \ucheby(\bff{v};\bff{w},\ruto),
    \end{align}
    %
    where $\ruto$ is a \textit{utopian point} satisfying $\ruto \succeq_P \bff{v}$ for all $\bff{v}\in\bfcl{V}$ and is a parameter of the algorithm.

\paragraph{Pareto CHMCTS (P-CHMCTS).}

    The notable exception to this form is Pareto MCTS \cite{pareto_momcts}, which uses a Pareto UCB algorithm \cite{pareto_ucb,pareto_ucb_2} at each decision node, and does not fit directly into the template of Equation (\ref{eq:5:prior_policy}). \textit{Pareto CHMCTS} (P-CHMCTS) is the algorithm that results from running Pareto UCB at each decision node within the CHMCTS framework.

    Pareto UCB adds a UCB-style bonus to each vector in the value set:
    \begin{align}
        \Vparetoconf(s) &= \cprune\left( \Vparetou(s) \right), \\
        \Vparetou(s) &= \left\{ 
            \left(\bff{q} + U_{\textnormal{pucb}}(s,a)\bff{1}, a \right)
            \mid a \in \cl{A}, \bff{q} \in \Qpareto(s,a)
        \right\}, \\
        U_{\textnormal{pucb}}(s,a) &= \bpareto \sqrt{\frac{\log(N(s))}{N(s,a)}}.
    \end{align}
    %
    where $\bpareto > 0$ is a bias parameter and where the notation $(\bff{q}, a)$ tags each vector with its corresponding action to allow the action to be recovered after the $\cprune$ operation.

    The search policy samples a point uniformly randomly from the \textit{confidence value set}, and selects the corresponding action. That is:
    \begin{align}
        \pipareto(a|s;\bff{w}) =  \sum_{(\bff{v},a')\in\Vparetoconf(s)} \frac{\one[a=a']}{|\Vparetoconf(s)|}.
    \end{align}

\subsection{Optimisations}
\label{sec:5-3-6-optimisations}

    Two practical optimisations are used in the implementation of CHMCTS: \textit{solved labelling}, which avoids wasting computation on well-explored parts of the search tree; and \textit{size pruning}, which limits the number of points stored in each convex hull value set.

\paragraph{Solved Labelling.}

    In planning, \textit{labelling} \cite{lrtdp} is used to mark nodes as solved and prune them from further consideration, avoiding wasted computation on parts of the tree that are already solved. A node is marked as solved when its value estimate has converged to the true value, which requires knowledge of the true transition distribution. In the setting considered here, the true transition distribution $p(\cdot|s,a)$ is unknown, so a probabilistic analogue for labelling is used instead.

    Each chance node $(s,a)$ is assigned an \textit{unsolved value} $\unsolved(s,a)\in[0,1]$, where $\unsolved(s,a)=0$ indicates that the node is solved and $\unsolved(s,a)=1$ indicates that it is unsolved. The solved value of a node is an upper bound on the probability that the node's value estimate has not yet converged.

    The unsolved value of a decision node is computed as the maximum unsolved value of its children:
    \begin{align}
        \unsolved(s) = \max_{a\in\cl{A}} \unsolved(s,a).
    \end{align}

    The unsolved values are updated during the backpropagation phase of \thtspp\ewe and used during the selection phase to filter out actions that have already been solved, to a tolerance of $\unsolvedtol$, with high probability. That is, at a decision node $s$, any action $a$ with $\unsolved(s,a) \leq \unsolvedtol$ is filtered out before selecting an action. The remainder of this section describes how the unsolved values are propagated through chance nodes, that is, how the value of $\unsolved(s,a)$ is computed.

    % Ideally $\unsolved(s,a)$ would be computed either as an expectation or maximisation over the successor states. Using the empirical distribution directly may be overly optimistic when few successor states have been observed, and using a maximisation over successor states penalises nodes with low-probability transitions that are rarely visited. An upper bound on the expected unsolved value is used instead, which accounts for both the estimation error in the empirical distribution and the probability mass of unobserved successor states.

    The solved value of a chance node $(s,a)$ is computed as an upper bound on the expected solved values of its successor states, using the empirical transition distribution $\frac{N(s')}{N(s,a)}$ as a proxy for the true distribution $p(s'|s,a)$. Since the true distribution is unknown, the bound must account for the estimation error in the empirical distribution. Let $\deltaunsolved > 0$ be a confidence parameter. The unsolved value $\unsolved(s,a)$ is defined as an upper bound satisfying:
    \begin{align}
        \Pr\left(\bb{E}_{s'\sim p(\cdot|s,a)}[\unsolved(s')] \leq \unsolved(s,a)\right) \geq 1-\deltaunsolved. \label{eq:5:unsolved_chance_defn}
    \end{align}

    In words, with probability at least $1-\deltaunsolved$, the expected solved value of the children of chance node $(s,a)$ is bounded by the upper bound $\unsolved(s,a)$.

    To derive this bound, the expected unsolved value is decomposed into contributions from observed and unobserved successor states:
    \begin{align}
        \mathbb{E}[\unsolved(s')] 
            =& \sum_{s'\in\sucobs{s}{a}} 
                p(s'|s,a)\unsolved(s') \label{eq:5:local6} \\
            & + \sum_{s'\notin\sucobs{s}{a}} 
                p(s'|s,a)\unsolved(s'), \label{eq:5:local7}
    \end{align}
    where $\sucobs{s}{a}= \{s'\in\suc{s}{a} \mid N(s') > 0\}$ is the set of successor states observed so far.

    For the observed successors, Corollary \ref{cor:4:bound_transition_distribution} of the DKW inequality (Theorem \ref{thrm:2:dkw_inequality}) gives with probability at least $1-\frac{\deltaunsolved}{2}$ for all $s'\in\sucobs{s}{a}$:
    \begin{align}
        p(s'|s,a) \leq \frac{N(s')}{N(s,a)} + 
            2\sqrt{\frac{\log(4/\deltaunsolved)}{2N(s,a)}},
    \end{align}
    so the contribution of observed successors in (\ref{eq:5:local6}) is bounded by:
    \begin{align}
        &\sum_{s'\in\sucobs{s}{a}} 
            p(s'|s,a)\unsolved(s') \\
        \leq & \sum_{s'\in\sucobs{s}{a}} 
            \left(\frac{N(s')}{N(s,a)} + 
            2\sqrt{\frac{\log(4/\deltaunsolved)}{2N(s,a)}}\right)\unsolved(s').
    \end{align}

    The \textit{Good-Turing estimate} (Section \ref{sec:2-5-convergence}) for the missing mass is:
    \begin{align}
        \goodturingestimate(s,a) 
            = \sum_{s'\in\suc{s}{a}}\frac{\one\left[N(s')=1\right]}{N(s,a)}.
    \end{align}

    For the unobserved successors, since $\unsolved(s')\leq 1$ for all $s'$, the contribution of unobserved successors (\ref{eq:5:local7}) is bounded by the missing mass using Theorem \ref{thrm:2:good_turing_bound} with probability at least $1-\frac{\deltaunsolved}{2}$:
    \begin{align}
        \sum_{s'\not\in\sucobs{s}{a}} p(s'|s,a) 
        \leq \goodturingestimate(s,a) + \sqrt{\frac{\log(2/\deltaunsolved)}{N(s,a)}}.
    \end{align}
    
    Combining these bounds via the union bound, the following assignment of $\unsolved(s,a)$ satisfies (\ref{eq:5:unsolved_chance_defn}):
    \begin{align}
        \unsolved(s,a) \leftarrow &
            \sum_{s'\in\sucobs{s}{a}} 
            \left(\frac{N(s')}{N(s,a)} + 
            2\sqrt{\frac{\log(4/\deltaunsolved)}{2N(s,a)}}\right)\unsolved(s') \\
            + &\ \goodturingestimate(s,a) + \sqrt{\frac{\log(2/\deltaunsolved)}{N(s,a)}}.
            \label{eq:5:solved_value}
    \end{align}
    
    When the $\unsolved$ labelling is used in CHMCTS, $\unsolvedtol$ and $\deltaunsolved$ are hyperparameters of the algorithm. A node is marked as solved when $\unsolved(s,a) \leq \unsolvedtol$, at which point it is excluded from further trials.

\paragraph{Convex Hull Size Pruning.}

    As the search progresses, the convex hull value sets at each node may grow large, increasing the computational cost of the $\cprune$ operation and the convex hull backups. To limit this cost, a maximum size $K_{\max}$ is imposed on each convex hull value set. When a value set exceeds this limit after a backup, points are pruned using the \textit{crowding distance} from the NSGA-II algorithm \cite{nsga_ii}

    The crowding distance of a point $\bff{v}$ in a set $\bfcl{V}$ measures how isolated $\bff{v}$ is from its neighbours in objective space. For each objective dimension $i=1,\ldots,D$, the points in $\bfcl{V}$ are sorted by their $i$th coordinate. The boundary points (those with the smallest and largest values in dimension $i$) are assigned an infinite crowding distance and are always retained. For interior points, the crowding distance contribution from dimension $i$ is:
    \begin{align}
        \textnormal{cd}_i(\bff{v}) = 
            \frac{v^+_i - v^-_i}{\max_{\bff{v}'\in\bfcl{V}} 
            v'_i - \min_{\bff{v}'\in\bfcl{V}} v'_i},
    \end{align}
    where $v^+_i$ and $v^-_i$ are the $i$th coordinates of the nearest neighbours of $\bff{v}$ above and below in dimension $i$ respectively. The total crowding distance is $\textnormal{cd}(\bff{v}) = \sum_{i=1}^D \textnormal{cd}_i(\bff{v})$. When the value set exceeds $K_{\max}$ points, the $K_{\max}$ points with the largest crowding distances are retained, preserving the spread of the convex hull.

\label{sec:5-3-7-entropy-convex-hulls}

    Convex hull value sets represent the set of achievable expected returns in \textit{objective space}, storing value vectors attainable by different policies. Entropy, by contrast, is a property of a policy rather than a value vector: it measures the uncertainty of the action distribution induced by a policy at each state.

    A key observation is that a single point on the convex hull does not correspond to a unique policy.
    % , as illustrated in Figure \ref{fig:5:entropy_convex_hull}. 
    Since the contextual policies considered in this chapter depend on the weight vector $\bff{w}$, a given value vector $\bff{v}\in\Valg(s)$ may be optimal for many different weight vectors, each inducing a different policy $\pi(\cdot|s;\bff{w})$ with a different entropy. 

    As a result, there is no principled way to associate a single entropy value with a point on the convex hull. This is the fundamental reason why the entropy-based exploration used in DENTS (Chapter \ref{ch:4-dents}) does not extend naturally to the multi-objective setting with convex hull value sets, and motivates the simplex map approach introduced in Chapter \ref{ch:6-simplexmaps}.

            An agent explores a gridworld, and aims to collect a piece of gold and a diamond and return to the starting location. There are three rewards, a time cost of $-1$ per timestep, a reward of $25$ for delivering a bottle and an additional cost of $-1$ for dropping a bottle. The trial ends when the agent has delivered two bottles.
    \end{description}

    For each environment, a horizon of $H=100$ was used where applicable.

\subsection{Evaluation Procedure} 
\label{sec:5-5-2-eval_procedure}

    It is worth noting that in multi-objective MCTS, the 
    quality of the convex hull at the root node is the 
    primary concern. Even when a multi-objective MCTS 
    algorithm is used in an online setting, where tree 
    search is interleaved with actions taken by the agent, 
    once a policy is selected at the initial state it 
    implicitly commits to a weighting over the objectives 
    (Section \ref{sec:5-1-intro}). A more efficient online 
    approach is therefore to run the multi-objective tree 
    search from the initial state, and subsequently follow 
    a single-objective MCTS algorithm in the scalarised MDP 
    with reward $R(s,a;\bff{w}) = \bff{w}^\top \bff{R}(s,a)$,
    where $\bff{w}$ is inferred from the choice made at the root node.

    To evaluate the quality of the convex hull at the root node, the \textit{Expected Utility Metric} (Definition \ref{def:2:eum}) and the \textit{Hypervolume} (Definition \ref{def:2:hypervolume}) are used.

    Given a context $\bff{w}$ and search tree $\cl{T}$ that defines a partial recommendation policy $\psi_{\alg}$, the policy will be made complete similarly to Section \ref{sec:4-4-2-eval_procedure}:
    \begin{align}
        \texttt{complete}(\psi_{\alg})(a|s;\bff{w}) =
        \begin{cases}
            \psi_{\alg}(a|s;\bff{w})                       & \text{ if } s\in\cl{T}, \\
            \frac{1}{|\cl{A}|}      & \text{ otherwise.}
        \end{cases} \label{eq:5:full_Recommend}
    \end{align}
    % 

    The Convex Hull (of policies) can then be written as 
    \begin{align}
        \Psi_{\alg} = \{\texttt{complete}(\psi_{\alg})\ |\ \bff{w}\in\Delta^D \}.
    \end{align}

    A Monte Carlo estimate of $\eum(\Psi_{\alg})$ (Definition \ref{def:2:eum}) is computed. Following \cite{well_spaced_points}, let $E(\Delta^D)$ be an evenly spaced set of points in $\Delta^D$. Then the \eum\ewe can be approximated as:
    \begin{align}
        \eum(\Psi_{\alg}) \approx \sum_{\bff{w}\in E(\Delta^D)} \bff{w}^\top \bff{\hat{V}}^{\bff{\tau}(\bff{w})},
    \end{align}
    %
    where $\bff{\tau}(\bff{w}) \sim \texttt{complete}(\psi_{\alg})$ is a sampled trajectory for the point $\bff{w}\in\Delta^D$, and $\bff{\hat{V}}^{\bff{\tau}(\bff{w})}$ is the cumulative return from the trajectory. That is, if the trajectory is $\bff{\tau}(\bff{w})=(s_0,a_0,\bff{r}_0,...,s_{h-1},a_{h-1},\bff{r}_{h-1},s_h)$, then $\bff{\hat{V}}^{\bff{\tau}(\bff{w})} = \sum_{i=0}^{h-1} \bff{r}_i$.

    Each algorithm is given $60$ seconds of search time, except for DST environments where $15$ seconds are given. Each algorithm uses $16$ threads for computation and value estimates for the EUM are computed using $128$ Monte Carlo samples, i.e. $|E(\Delta^D)| = 128$.

    As in Section \ref{sec:4-4-results}, the BayesOpt package \cite{bayesopt} is used to tune the hyperparameters of all algorithms. Full details on the hyperparameter tuning of multi-objective algorithms is given in Appendix \ref{app:expr-deets-ch5}.

\subsection{The Fundamental Assumption of MOMCTS}
\label{sec:5-6-2-fundamental_assumption_of_momcts}

    This chapter has discussed that following locally optimal actions does not always lead to globally optimal policies in the multi-objective setting. This section formalises this result and shows that locally optimal actions lead to globally optimal policies if and only if the utility function is linear.

    Because MCTS methods are built on top of making local decisions at each node, this implies that MOMCTS, for \textit{arbitrary} MOMDPs, cannot guarantee convergence to optimal policies unless the utility function is linear. The MOMDP used to show this result is \textit{stochastic}, shown in Figure \ref{fig:5:momdp_proof}. Prior work in MOMCTS (Section \ref{sec:3-5-momcts}) has typically focused on finding Pareto fronts for \textit{deterministic} MOMDPs. It is conjectured here that stochasticity of the MOMDP is a 
    necessary condition for this result to hold.


    \begin{figure}
        \centering
        \includegraphics[width=\textwidth]{figures/ch5/proof.pdf}
        \caption[A stochastic MOMDP where making locally optimal decisions does not yield globally optimal policies for \textit{Chebyshev} utilities.]{A stochastic MOMDP where making locally optimal decisions does not yield globally optimal policies for \textit{Chebyshev} utilities.}
        \label{fig:5:momdp_proof}
    \end{figure}

    Consider an arbitrary MOMDP $\bfcl{M}$, utility function $u$ and set of policies $\Pi$. The optimal value sets are defined as:
    \begin{align}
        \bfcl{V}^*_u(s) &= \{\bff{V}^{\pi}(s) \mid \pi \in U(\Pi;u)\}, \\
        \bfcl{Q}^*_u(s, a) &= \{\bff{Q}^{\pi}(s, a) \mid \pi \in U(\Pi;u)\}.
    \end{align}

    Then the locally optimal policy is defined as:
    \begin{align}
        \pi_{\text{local}}(s; \bff{w}) \in \argmax_{a \in \cl{A}}\;
            \max_{\bff{q} \in \bfcl{Q}^*_u(s, a)} u(\bff{q}; \bff{w}).
    \end{align}

    \begin{theorem}
        \label{thm:per-step-linear}
        The policy $\pi_{\text{local}}(\cdot; \bff{w})$ is utility-optimal (i.e. $u(\bff{V}^{\pi_{\text{local}}}(s_0); \bff{w}) = u(\bff{V}^{\pi^{\bff{w}}_u}(s_0); \bff{w})$) for every MOMDP $\bfcl{M}$ and every $\bff{w} \in \Delta^D$ if and only if $u(\cdot; \bff{w})$ is linear in its first argument for every
        $\bff{w} \in \Delta^D$.
    \end{theorem}
    \begin{proofoutline}
        \textbf{If direction.}

        Follows from the convergence of Convex Hull Value Iteration \cite{chvi}.

        \textbf{Only if direction.}

        The contrapositive is established by exhibiting a single MOMDP, utility function and weight vector where the locally optimal policy $\pi_{\text{local}}(\cdot; \bff{w})$ is not utility-optimal. 
        
        Let $\bfcl{M}$ be the MOMDP in Figure \ref{fig:5:momdp_proof} and $u$ be the Chebyshev utility with reference point $\bff{z} = (6, 6)$, defined as:
        \begin{align}
            u(\bff{v}; \bff{w}) = -\max_i w_i \left| v_i - z_i \right|.
        \end{align}

        The four paths through the MOMDP yield rewards of:
        %
        \begin{align*}
            s_0 \to \sss{1} \to \aaa{0} &: (5, 0), &
            s_0 \to \sss{1} \to \aaa{1} &: (2, 2), \\
            s_0 \to \sss{2} \to \aaa{0} &: (2, 2), &
            s_0 \to \sss{2} \to \aaa{1} &: (0, 5).
        \end{align*}

        Fix the weight vector $\bff{w} = (\tfrac{1}{2}, \tfrac{1}{2})$. By inspection, there are four stationary policies yielding expected values at $s_0$ of $(3.5, 1)$, 
        $(2.5, 2.5)$, $(2, 2)$, and $(1, 3.5)$, with the policy achieving 
        $(2.5, 2.5)$ being utility-optimal for $\bff{w}$:
        %
        \begin{equation*}
            u((3.5, 1); \bff{w}) = -2.5, \quad
            u((2.5, 2.5); \bff{w}) = -1.75, \quad
            u((1, 3.5); \bff{w}) = -2.5.
        \end{equation*}
        %
        (The value $(2, 2)$ is Pareto-dominated and is omitted.) The utility-optimal 
        policy selects $\aaa{0}$ at $\sss{1}$ (receiving $(5, 0)$) and $\aaa{1}$ at $\sss{2}$ 
        (receiving $(0, 5)$), giving expected value $(2.5, 2.5)$ and utility $-1.75$.
    
        Now consider $\pi_{\text{local}}$. The Pareto fronts at the successor 
        states are singletons,
        %
        \begin{align*}
            \bfcl{Q}^*_u(\sss{1}, \aaa{0}) &= \{(5, 0)\}, &
            \bfcl{Q}^*_u(\sss{1}, \aaa{1}) &= \{(2, 2)\}, \\
            \bfcl{Q}^*_u(\sss{2}, \aaa{0}) &= \{(2, 2)\}, &
            \bfcl{Q}^*_u(\sss{2}, \aaa{1}) &= \{(0, 5)\},
        \end{align*}
        %
        and the Chebyshev utilities are $u((5, 0); \bff{w}) = u((0, 5); \bff{w}) = -3$ 
        and $u((2, 2); \bff{w}) = -2$. Hence $\pi_{\text{local}}$ selects $\aaa{1}$ at 
        $\sss{1}$ and $\aaa{0}$ at $\sss{2}$, both yielding $(2, 2)$. The expected value at $s_0$ is
        %
        \begin{equation*}
            \bff{V}^{\pi_{\text{local}}}(s_0) 
            = \tfrac{1}{2}(2, 2) + \tfrac{1}{2}(2, 2) = (2, 2),
        \end{equation*}
        %
        with utility $u((2, 2); \bff{w}) = -2 < -1.75$. Therefore 
        $\pi_{\text{local}}$ is not utility-optimal.
    \end{proofoutline}

    The issue with the MOMDP defined in Figure \ref{fig:5:momdp_proof} is that the optimal policy requires optimising towards the conflicting objectives of $(5, 0)$ and $(0, 5)$ at the same time, depending on the outcome of the stochastic transition to $\sss{1}$ or $\sss{2}$. It can additionally be shown that for any utopian point $\bff{z}$ and weight $\bff{w}$ the reward of $(2,2)$ will be locally optimal for one of the states $\sss{1}$ or $\sss{2}$ and the locally optimal policy $\pi_{\text{local}}$ will not be utility optimal.











\subsection{Convergence of CHMCTS}
\label{sec:5-6-3-convergence_of_chmcts}


    This section outlines how the convergence of CHMCTS can be proven. First, if all states are visited infinitely often, then by the convergence of Convex Hull Value Iteration \cite{chvi} the convex hull value sets converge to the convex coverage sets. 

    \begin{theorem}
        For any MOMDP $\bfcl{M}$, if $N(s)\rap\infty$ and $N(s,a)\rap\infty$ for all $s\in\cl{S}$ and $a\in\cl{A}$, then the convex hull value sets $\Valg(s)\rap\bfcl{V}^*_u(s)$ and $\Qalg(s,a)\rap\bfcl{Q}^*_u(s,a)$ for all $s\in\cl{S}$ and $a\in\cl{A}$, where $\bfcl{V}^*_u(s)$ and $\bfcl{Q}^*_u(s,a)$ are the optimal value sets for the linear utility function $u$.
    \end{theorem}
    \begin{proofoutline}
        Follows from the convergence of Convex Hull Value Iteration \cite{chvi}.
    \end{proofoutline}

    By Lemma \ref{lem:inf_often_action_select} it then follows that for CH-BTS that the convex hull value sets converge to the optimal value sets. Lemma \ref{lem:inf_often_action_select_uct} is the corresponding lemma for UCT-based policies:

    \begin{lemma} 
        \label{lem:inf_often_action_select_uct}
        Let $\rho^m$ be an arbitrary policy and consider the following search policy used on the $m$th round of a bandit problem:
        \begin{align} 
            \pi^m(\bff{w}) = \argmax_{a\in\cl{A}} \dot{Q}(a) + b \sqrt{\frac{\log(m)}{M^m(a)}},
        \end{align}
        where $b > 0$ is a bias parameter, $M^m(a)$ is the number of times $a$ was selected in the first $m$ rounds, and $\dot{Q}$ is an arbitrary function with $\dot{Q}(a) \leq Q_{\max}$ for all $a\in\cl{A}$. Then $M^m(a)\rightarrow\infty$ as $m\rightarrow\infty$ for all $a\in\cl{A}$.
    \end{lemma}
    \begin{proofoutline}
        By contradiction: if $M^m(a)$ remained bounded while every other action's count grew, the $b\sqrt{\log(m)/M^m(a)}$ term would diverge and force $a$ to be selected, contradicting boundedness.
    \end{proofoutline}

    Given Lemmas \ref{lem:inf_often_action_select} and \ref{lem:inf_often_action_select_uct} it then follows that for all CHMCTS algorithms defined in Section \ref{sec:5-3-chmcts} that the convex hull value sets converge to the optimal value sets. Hence, by Theorem \ref{thm:per-step-linear} it follows that for any CHMCTS algorithm $\alg$ that $\bb{E}\left[\ctxsreg(s,\psi_{\alg})\right] \rightarrow 0$ as the number of trials $n\rightarrow\infty$.

\section{Summary}
\label{sec:5-7-summary}

    This chapter extended the MCTS framework to the 
    multi-objective decision support scenario with linear 
    utilities, introducing two approaches for contextual 
    planning in MOMDPs.

    The consistent action selection problem was identified 
    as a fundamental challenge in MOMCTS (Section 
    \ref{sec:5-1-intro}): actions taken within a trial 
    should be consistent with each other (or all actions consistent with a single weight vector), 
    otherwise the search policy optimises for no 
    particular objective. Contextual tree search was 
    proposed as a solution, where a weight vector 
    $\bff{w}$ is sampled at the start of each trial and 
    used to guide all subsequent action selections. It 
    was also shown (Theorem \ref{thm:per-step-linear}) 
    that locally optimal action selection yields globally 
    optimal policies if and only if the utility function 
    is linear, confirming that the assumption of linear utility is 
    necessary for a MOMCTS algorithm to guarantee 
    convergence, at least for stochastic environments, addressing \stochq\ewe theoretically.

    \textit{Contextual Zooming for Trees} (CZT) was 
    introduced (Section \ref{sec:5-2-czt}) as a natural 
    contextual analogue of UCT, applying the Contextual 
    Zooming algorithm at each decision node. 
    \textit{Convex Hull MCTS} (CHMCTS) was introduced 
    (Section \ref{sec:5-3-chmcts}) as a generalisation 
    of prior MOMCTS work to stochastic environments, 
    maintaining convex hull value sets at each node and 
    updating them via convex hull backups. The prior 
    MOMCTS algorithms of Chapter \ref{ch:3-relatedwork} 
    were recovered as special cases of the CHMCTS 
    framework by substituting appropriate 
    context-independent search policies.

    Empirically, CHMCTS algorithms generally outperformed 
    CZT on deterministic and near-deterministic 
    environments, where the convex hull value sets 
    provide an accurate representation of achievable 
    returns. However, in stochastic environments the 
    computational cost of the convex hull backups grows 
    rapidly, reducing the number of trials that can be 
    run within a fixed time budget and limiting 
    performance. CZT avoids this issue but exhibits the 
    exploitation bias of UCT-style algorithms, converging 
    to locally optimal solutions. Leaving \stochq\ewe to be unsatisfactorily addressed empirically. However, \moscalabilityq\ewe is addressed in part by the scalability results in Section \ref{sec:5-5-4-scalability}.

    Two limitations of the convex hull approach motivate 
    the work of Chapter \ref{ch:6-simplexmaps}. First, 
    convex hull value sets become computationally expensive in stochastic 
    environments, suggesting that a hierarchical  
    representation would may more scalable. Second, 
    convex hull value sets do not naturally support entropy-based exploration, making it difficult to utilise entropy as an exploration bonus. Both 
    limitations are addressed by the simplex map 
    approach introduced in Chapter 
    \ref{ch:6-simplexmaps}.
