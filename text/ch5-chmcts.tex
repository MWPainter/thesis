\chapter{\label{ch:5-chmcts}Convex Hull Monte Carlo Tree Search}

    \minitoc

    This chapter considers multi-objective Monte Carlo Tree Search 
    (MOMCTS) for the decision support scenario (\todo{ref back to ch2 fig}) with linear utilities, where the agent must plan for an unknown weight vector $\bff{w}$ over the objectives. Two approaches are considered in this chapter, the first following the same design paradigm as UCT where an algorithm for CMAB problems (Section \ref{sec:2-1-3-contextual-mab}) is applied at each decision node; the second is to maintain a convex hull value set at each node in the 
    search tree, representing the set of achievable expected returns 
    across all possible weightings over the objectives which is updated using convex hull backups (Section \ref{sec:2-6-1-chvi}). 
    
    This chapter is organised as follows:
    \begin{itemize}
        \item Section \ref{sec:5-1-intro} motivates the 
            multi-objective decision making framework used in this 
            chapter, and generalises the notions of simple and cumulative regret to 
            \textit{contextual simple regret} and 
            \textit{contextual cumulative regret} for the multi-objective setting;
        \item Section \ref{sec:5-2-czt} introduces 
            \textit{Contextual Zooming for Trees} (CZT), a 
            MOMCTS algorithm following the same design paradigm 
            as UCT;
        \item Section \ref{sec:5-3-chmcts} generalises prior work 
            in MOMCTS to stochastic environments through 
            \textit{convex hull backups};
        \item Section \ref{sec:5-5-results} presents empirical 
            results on Deep Sea Treasure environments and a range 
            of multi-objective environments from the 
            Multi-Objective Gymnasium library \todo{cite?};
        \item Section \ref{sec:5-6-theory} provides a brief 
            theoretical discussion of the MOMCTS algorithms 
            introduced in this chapter.
    \end{itemize}

\section{Introduction and Motivation}
\label{sec:5-1-intro}

    This section motivates the use of \textit{contextual tree search} 
    for planning in multi-objective settings, and highlights why 
    consistent action selection with respect to a fixed objective 
    is a necessary component of effective MOMCTS. The notions of 
    cumulative and simple regret are then generalised to the 
    multi-objective setting.

\subsection{Consistent Action Selection}

    \begin{figure}
        \centering
        \includegraphics[width=0.6\textwidth]{figures/todo.jpg}
        \caption[A motivating example MOMDP for contextual tree search.]{Tree 
        MOMDP, with two actions at the root. From the first action, rewards of 
        $(1,0)$, $(0.5,0.5)$ and $(0,0.75)$ are available. From the second, 
        rewards of $(0.75,0)$, $(0.5,0.5)$ and $(0,1)$ are available. The 
        convex hull value set is formed of $\{(1,0),(0,1)\}$.}
        \label{fig:5:motivating_example}
    \end{figure}

    Consider the MOMDP illustrated in Figure \ref{fig:5:motivating_example}, 
    with two objectives. The convex hull value set of the MOMDP is $\{(1,0),(0,1)\}$: the reward $(1,0)$ is only reachable via the left action, and $(0,1)$ only via the right action. The optimal action at the root node therefore depends entirely on the weight vector $\bff{w}$: if $w_1 > w_2$ then the left action is optimal, and if $w_2 > w_1$ then the right action is optimal.

    Once the agent has taken the left action from the root node, it has implicitly committed to optimising towards the first objective (i.e. $w_1 > w_2$) because the rewards available from the right action are no longer reachable. Although all actions at the immediate child nodes are \textit{locally optimal} (i.e. they lead to a value in the local convex hull value set), only the action leading to $(1,0)$ is consistent with the implicit commitment made at the root node.

    \begin{figure}
        \centering
        \includegraphics[width=0.6\textwidth]{figures/todo.jpg}
        \caption[A motivating example MOMDP for contextual tree search.]{\todo{update caption with }}
        \label{fig:5:motivating_vanmoffaert}
    \end{figure}

    The problem of \textit{consistent action selection} is also discussed by \todo{citet vanmoffaert14a}. They compute and track Pareto-optimal policies using an adaptation of Q-learning and highlight that even if an action is \textit{locally optimal}, it may not be \textit{globally optimal}. To demonstrate this, they use an example with three optimal paths (Figure \ref{fig:5:motivating_vanmoffaert}), where the black path alternates between multiple optimal paths, taking only locally optimal actions at each step but fails to follow any single globally optimal path.

\subsection{Contextual Tree Search}
\label{sec:5-1-1-contextual_tree_search}

    \todo{citet vanmoffaert and double check} solve the consistent action selection problem by explicitly tracking the optimal policies across the full MOMDP. The approach taken in this thesis is to consider a \textit{contextual tree search}, where a context weight vector $\bff{w}$ is sampled at the start of each trial and used to guide action selection throughout that trial. In the context of MORL with linear utilities, this is sufficient to resolve the consistent action selection problem, ensuring that all actions within a trial are consistent with a single objective.

    In the \thtspp\ewe framework, the context weight $\bff{w}$ is sampled during the \samplecontext\ewe function (Section \ref{sec:2-4-1-thts}) and passed to all subsequent steps of the trial, including the search policy and backup functions.

    \begin{defn}
        \label{def:5:context_policy}
        A \textnormal{contextual policy} $\pi(a|s;\bff{w})$ is a 
        policy that may depend on the context weight vector 
        $\bff{w}\in\Delta^{D}$, where $\pi(a|s;\bff{w})$ gives 
        the probability of selecting action $a$ at state $s$ under 
        weight $\bff{w}$.

        A policy $\pi$ is \textnormal{context independent} if 
        $\pi(a|s;\bff{w}) = \pi(a|s;\bff{w}')$ for all 
        $\bff{w},\bff{w}'\in\Delta^{D}$ and all $s\in\cl{S}$, 
        $a\in\cl{A}$. Otherwise, $\pi$ is 
        \textnormal{context dependent}.
    \end{defn}

    To reason about the performance of a contextual tree search algorithm, the notions of cumulative and simple regret are generalised to \textit{contextual cumulative regret} and \textit{contextual simple regret}. These definitions reduce to the standard definitions (Definitions \ref{def:2:cumulative_regret} and \ref{def:2:simple_regret}) when $D=1$.

    \begin{defn}
        \label{def:5:contextual_cumulative_regret}
        Let $\{\bff{w}^i\}_{i=1}^n$ be a sequence of weight vectors sampled uniformly from $\Delta^D$. The \textnormal{contextual cumulative regret} of the search policies $\pi^{1:n} = (\pi^1,\ldots,\pi^n)$ after $n$ trials is:
        \begin{align}
            \ctxreg(s,\pi^{1:n}) = 
                \sum_{i=1}^n \left( 
                    \max_{\pi'} {\bff{w}^i}^\top \bff{V}^{\pi'}(s)
                    - {\bff{w}^i}^\top \bff{V}^{\pi^i}(s)
                \right).
        \end{align}
    \end{defn}
    
    \begin{defn}
        \label{def:5:contextual_simple_regret}
        The \textnormal{contextual simple regret} of a recommendation policy $\psi$ at state $s\in\cl{S}$ is:
        \begin{align}
            \ctxsreg(s,\psi) = \bb{E}_{\bff{w}\sim\Delta^D}\left[
                \max_{\pi'} {\bff{w}}^\top \bff{V}^{\pi'}(s) 
                - {\bff{w}}^\top \bff{V}^{\psi(\cdot;\bff{w})}(s)
            \right],
        \end{align}
        where $\bff{w}\sim\Delta^D$ denotes a weight vector sampled uniformly from the $D$-dimensional unit simplex.
    \end{defn}

    Several prior MOMCTS works \todo{cite}, discussed in Section \todo{ref ch3}, use a context-independent search policy. It is shown in Section \ref{sec:5-6-theory} that any context-independent search policy suffers linear contextual cumulative regret, and any context-independent recommendation policy suffers linear contextual simple regret.

\section{Contextual Zooming for Trees}
\label{sec:5-2-czt}

    \begin{figure}
        \centering
        \includegraphics[width=0.5\textwidth]{figures/todo.jpg}
        \caption[Design paradigm for CZT.]{The design paradigm 
        for CZT. UCB is lifted to UCT by applying it at each node 
        of the search tree. CZ generalises UCB to contextual bandits. 
        CZT follows both directions, applying CZ at each node of 
        the search tree.}
        \label{fig:5:czt_diagram}
    \end{figure}

    A common design paradigm in MCTS is to adapt a MAB algorithm to run at each decision node of the search tree, treating the node as a (non-stationary) bandit problem. UCT follows this paradigm by running UCB at each node (Section \ref{sec:2-4-2-uct}). 
    
    The \textit{Contextual Zooming} (CZ) algorithm \cite{cz,cz_long} is a generalisation of the UCB algorithm to contextual bandits (Section \ref{sec:2-1-3-contextual-mab}). Following the same methodology, \textit{Contextual Zooming for Trees} (CZT) runs the CZ algorithm at each decision node of the search tree. The relationship between these algorithms is illustrated in Figure \ref{fig:5:czt_diagram}.
    
    CZT is presented in this section in the context of MOMCTS, where the context is the weight vector $\bff{w}$ sampled at the start of each trial. Although CZ, and by extension CZT, is not motivated by the exploration setting this thesis focuses on, it is introduced as a natural contextual analogue of UCT, and as a contextual search policy baseline for MOMCTS, where no such baseline currently exists.

\subsection{Algorithm Definition}
\label{sec:5-2-1-czt}

    Contextual Zooming operates over a similarity space (defined in Section \ref{sec:2-1-3-contextual-mab}), which in the case of CZT is the weight-action space $\Delta^D\times\cl{A}$. The following metric is defined over the weight-action space for use in CZT:
    %
    \begin{align}
        \dczt((\bff{w},a),(\bff{w}',a')) = \| \bff{w} - \bff{w}' \|_2 + C_{\textnormal{czt}}\one\left[a\neq a'\right], \label{eq:5:czt_metric}
    \end{align}
    %
    where $\Cczt>\sqrt{2}$ is a large constant such that the distance between any two actions is always greater than the distance between any two weights.

    CZ maintains a set of \textit{active balls} that cover the weight-action space. 
    In CZT, let $A(s,a)$ denote the set of active balls for action $a$ at decision node $s$, where a ball with centre $(\bff{w},a)$ and radius $r$ is defined as $\ball((\bff{w},a),r) = \{(\bff{w}',a')\in\Delta^D\times\cl{A} \mid d((\bff{w},a),(\bff{w}',a')) < r\}$. Let $A(s)$ be the set of all active balls at decision node $s$, i.e. $A(s) = \cup_{a\in\cl{A}} A(s,a)$.

    Each set of active balls is initialised to a single ball, using the centroid of the simplex $\frac{1}{D}\bff{1}$, where $\bff{1}$ is the vector of all ones. Specifically, 
    \begin{align}
        A^0(s,a) = \left\{ \ball\left(\left(\frac{1}{D}\bff{1},a\right),\sqrt{\frac{D-1}{D}}\right) \right\}. \label{eq:5:czt_initial_ball}
    \end{align}

    Each ball $B\in A(s,a)$ has an associated visit count $M(B)$ and average return value estimate $\Qczt(B)$. The visit count and value estimates are initialised to $M(B)=0$ and $\Qczt(B)=\bff{0}$ respectively.

    Two rules are applied to run contextual zooming: 
    \begin{itemize}
        \item the \textit{selection rule}, to select a ball (and action) from the set of active balls at the current decision node, as part of the \selectaction\ewe function;
        \item the \textit{activation rule}, which optionally adds a ball with smaller radius to the set of active balls, as part of the \backupv\ewe function.
    \end{itemize}
    
\paragraph{Selection Rule}

    Let $\bff{w}$ be the context weight sampled for the $n$th trial, let $r(B)$ denote the radius of a ball $B\in A(s,a)$. The \textit{domain} of $B$ in CZ consists of the portion of the ball not covered by any smaller active ball. Specifically,
    \begin{align}
        \dom(B) = B - \left( \bigcup_{B'\in A(s,a):r(B')<r(B)} 
            B' \right). \label{eq:5:czt_domain}
    \end{align}

    The \textit{confidence radius} $\conf(B)$ of ball $B$ is:
    \begin{align}
        \conf(B) = \bczt \sqrt{\frac{\log(N(s))}{1+M(B)}},
            \label{eq:5:czt_conf_radius}
    \end{align}
    where $\bczt>0$ is a UCB style bias parameter.

    Let $\relevant(s,a;\bff{w})$ denote the set of \textit{relevant} balls at a decision node $s$ for action $a$. A ball is relevant if $(\bff{w},a)$ is in the domain of the ball. Specifically,
    \begin{align}
        \relevant(s,a;\bff{w}) = \{ B \in A(s,a) \mid (\bff{w},a)\in\dom(B)\}, \\
        \relevant(s;\bff{w}) = \cup_{a\in\cl{A}} \relevant(s,a;\bff{w}).
    \end{align}

    Contextual Zooming defines a confidence interval term, called the \textit{pre-index}, around each ball, similar to UCB's confidence interval term:
    \begin{align}
        \Ipreczt(B;\bff{w}) = \bff{w}^\top \Qczt(B) + r(B) + \conf(B). \label{eq:5:cz_pre_index}
    \end{align}

    The full \textit{index} of a ball $B$ allows a newly activated ball with few visits to inherit an optimistic value estimate from nearby well-explored balls, ensuring it remains eligible for selection. Specifically,
    \begin{align}
        \Iczt(B;\bff{w}) = 
            r(B) + \min_{B'\in A(s)} \left( \Ipreczt(B';\bff{w}) + d(B,B') \right), \label{eq:5:cz_index}
    \end{align}

    where $d(B,B')$ is the distance between the centres of balls $B$ and $B'$ according to the metric $d$. In CZT the constant $\Cczt$ is set to a large enough value such that balls with a different action will never minimise the expression, so it suffices to consider only balls with the same action as $B$ in Equation (\ref{eq:5:cz_index}).

    To select a ball, the \selectball\ewe function chooses the ball from the set of relevant balls with the highest index:
    \begin{align}
        \selectball(s,\bff{w}) = \argmax_{B\in\relevant(s;\bff{w})} \Iczt(B).
    \end{align}

    If $\selectball(s,\bff{w}) = \ball((\bff{w}_B,a_B),r_B)$, then the CZT selection policy returns $\piczt(s;\bff{w})=a_B$.

\paragraph{Backups and Activation Rule}

    Let $\bff{\tau}_{:h}\sim\piczt$ be the trajectory sampled in the selection phase of CZT. Let $B_t$ be the ball selected at the decision node $s_t$. 

    The backup functions update the ball selection counts and the multi-objective average returns for $t=h-1,\ldots,0$:
    \begin{align}
        M(B_t) &\leftarrow M(B_t) + 1, \\
        \Qczt(B_t) &\leftarrow \frac{M(B_t)-1}{M(B_t)} \Qczt(B_t) + \frac{\Vinitbff(s_h) + \sum_{i=t}^{h-1} \bff{r}_i}{M(B_t)},
    \end{align}
    where $\Vinitbff(s_h)$ is the (multi-objective) heuristic value function.

    The \textit{activation rule} introduces a new ball with smaller radius when the following conditions are met:
    \begin{align}
        M(B_t) > \Mmin \quad \text{and} \quad \conf(B_t) \leq r(B_t),
    \end{align}
    %
    where $\Mmin$ is a minimum selection threshold, and when both conditions are met, a new ball $B_{\textnormal{new}} = \ball((\bff{w},a_t),\frac{r(B_t)}{2})$ is added to the set of active balls $A(s,a)$.

\paragraph{Recommendation Policy}

    To recommend an action in CZT, the ball with maximum linear utility from the set of active balls is selected. Specifically,
    \begin{align}
        \psiczt(s;\bff{w}) = \argmax_{a\in\cl{A}} \max_{B\in \relevant(s,a;\bff{w})} \bff{w}^\top \Qczt(B).
    \end{align}


\subsection{Differences from Contextual Zooming}
\label{sec:5-2-2-cz-diffs}

    Section \ref{sec:5-2-1-czt} described the CZT algorithm. The main adaptations from CZ are summarised below:
    \begin{enumerate}
        \item 
            In CZT, the set of active balls is specialised to a set for each action. In CZ, a single set of active balls is used across the entire \textit{similarity space} (defined in Section \ref{sec:2-1-3-contextual-mab}).
        \item \label{item:5:local1}
            CZ assumes that the algorithm is run for a known number of rounds $T$, and uses this value instead of the current number of visits at the decision node $N(s)$ in the confidence radii (Equation \ref{eq:5:cz_conf_radius}).
        \item 
            The activation rule of CZ does not have a hard lower bound on the number of ball visits required to add a new child ball. CZT makes this adaptation to account for item \ref{item:5:local1}, which results in smaller confidence radii.
        \item 
            In CZT, all balls contain exactly one action, whereas in CZ with a general continuous space of arms, balls cover many arms. After a ball is selected in CZ, an arm is sampled uniformly randomly from the domain of the selected ball. 
    \end{enumerate}

    CZ \todo{cite} extends its regret analysis from a fixed number of rounds $T$ to an unknown number of rounds using the \textit{doubling trick} \cite{doubling_trick}. The doubling trick runs an independent instance of the algorithm for $T$ rounds, then $2T$ rounds, then $4T$ rounds and so on. In practice, however, discarding all accumulated data at each doubling interval is undesirable. Instead, in CZT the number of visits $N(s)$ at the current node is used directly in place of $T$, which avoids the need for the doubling trick at the cost of losing regret guarantees.

\section{Convex Hull Monte Carlo Tree Search}
\label{sec:5-3-chmcts}

    This section introduces \textit{Convex Hull Monte Carlo Tree Search} (CHMCTS), which generalises prior work in MOMCTS \todo{ref ch3} to stochastic environments by maintaining \textit{convex hull value sets} at each node in the search tree and updating them using convex hull backups (Section \ref{sec:2-6-1-chvi}). When the MOMDP is deterministic and a search policy from prior work is used, CHMCTS reduces to the prior MOMCTS algorithms discussed in Chapter \ref{ch:3-litrev}.

    The common \thtspp\ewe framework and backup equations shared by all CHMCTS algorithms are given first. The search policies for each algorithm are then defined. Two practical optimisations are discussed in Section \ref{sec:5-6-optimisations}, and the section concludes with a brief discussion on the relationship between entropy and convex hull value sets.

\subsection{Common Framework}
\label{sec:5-3-1-framework}

    Let $\alg$ denote any of the CHMCTS algorithms defined in this section. At each decision node $s$, a convex hull value set $\Valg(s)$ is maintained, and at each chance node $(s,a)$, a convex hull Q-value set $\Qalg(s,a)$ is maintained. The specific form of the search policy varies between algorithms and is defined in the subsections below.

    Given a context weight $\bff{w}\in\Delta^D$ and a trajectory $\tau_{:h}=(s_0,a_0,\bff{r}_0,\ldots,s_{h-1},a_{h-1},\bff{r}_{h-1},s_h)\sim\pialg(\cdot|\cdot;\bff{w})$, the value set estimates are updated for $t=h-1,\ldots,0$:
    \begin{align}
        \Valg(s_t) &\leftarrow \cprune \left( 
            \bigcup_{a\in\cl{A}} \Qalg(s_t,a) \right), 
            \label{eq:5:chmcts_v_backup} \\
        \Qalg(s_t,a_t) &\leftarrow \bff{R}(s_t,a_t) + 
            \sum_{s'\in\suc{s_t}{a_t}} 
            \frac{N(s')}{N(s_t,a_t)}\Valg(s'). 
            \label{eq:5:chmcts_q_backup}
    \end{align}

    The $\cprune$ operation, defined in Equation 
    (\ref{eq:2:cprune_def}) and repeated here for convenience, 
    retains only those vectors in a set $\bfcl{V}$ that are 
    optimal for at least one weight vector:
    \begin{align}
        \cprune(\bfcl{V}) = \{\bff{v}\in\bfcl{V} \mid 
            \exists \bff{w}\in\Delta^D.\ 
            \forall \bff{v}'\in\bfcl{V}-\{\bff{v}\}.\ 
            \bff{w}^\top \bff{v} > \bff{w}^\top \bff{v}' \}. 
            \label{eq:5:cprune_def}
    \end{align}

    Since the context weight is passed to policies, it is useful to define \textit{contextual value estimates}, which extract a scalar value from a convex hull value set for a given weight $\bff{w}\in\Delta^D$:
    \begin{align}
        \Valgctx{\bff{w}}(s) &= \max_{\bff{v}\in\Valg(s)} 
            \bff{w}^\top \bff{v}, 
            \label{eq:5:contextual_value} \\
        \Qalgctx{\bff{w}}(s,a) &= \max_{\bff{q}\in\Qalg(s,a)} 
            \bff{w}^\top \bff{q}. 
            \label{eq:5:contextual_qvalue}
    \end{align}

    The recommendation policy selects the action with the highest contextual Q-value under the given weight:
    \begin{align}
        \psialg(s;\bff{w}) = \argmax_{a\in\cl{A}} 
            \Qalgctx{\bff{w}}(s,a). \label{eq:5:chmcts_rec}
    \end{align}

\subsection{Convex Hull UCT}
\label{sec:5-3-2-chuct}

    Another natural extension of UCT to the contextual multi-objective setting is to replace the scalar Q-value estimates (used in Section \ref{sec:2-4-2-uct}) with contextual value estimates from Equations (\ref{eq:5:contextual_value}) and (\ref{eq:5:contextual_qvalue}). The resulting algorithm, \textit{Convex Hull UCT} (CH-UCT), uses the following search policy:
    \begin{align}
        \pichuct(s;\bff{w}) = \argmax_{a\in\cl{A}} \left[
            \Qchuctctx{\bff{w}}(s,a) + \cchuct 
            \sqrt{\frac{\log(N(s))}{N(s,a)}} \right],
            \label{eq:5:chuct_policy}
    \end{align}
    where $\cchuct > 0$ is a bias parameter controlling exploration. The search policy is context-dependent, since the contextual Q-value $\Qchuctctx{\bff{w}}(s,a)$ depends on $\bff{w}$, ensuring that action selection is consistent with the sampled objective throughout each trial.

\subsection{Convex Hull BTS}
\label{sec:5-3-3-chbts}

    \textit{Convex Hull BTS} (CH-BTS) uses the BTS search policy (Section \ref{sec:4-2-1-bts}) within the CHMCTS framework, using contextual value estimates in place of the scalar Bellman value estimates:
    \begin{align}
        \rhochbts(a|s;\bff{w}) &\propto \exp\left(
            \frac{\Qchbtsctx{\bff{w}}(s,a)}
            {\alphachbts(N(s))}\right), 
            \label{eq:5:chbts_rho} \\
        \pichbts(a|s;\bff{w}) &= (1-\lambdachbts)
            \rhochbts(a|s;\bff{w}) 
            + \frac{\lambdachbts}{|\cl{A}|}, 
            \label{eq:5:chbts_policy} \\
        \lambda(s,\epsilon) &= \min\left(1, 
            \frac{\epsilon}{\log(e+N(s))}\right),
    \end{align}
    where $\alphachbts(x) > 0$ is a temperature function and $\epschbts \in [0,\infty)$ is an exploration parameter. As in BTS, the temperature function controls the trade-off between exploration and exploitation, and the exploration parameter ensures that all actions are eventually explored.

\subsection{Convex Hull CZT}
\label{sec:5-3-4-chczt}

    In single-objective MCTS, Bellman backups often outperform average return backups in the exploration setting (Chapter \ref{ch:4-dents}), since they propagate value information more directly through the tree by bootstrapping estimates from successor states rather than averaging over full trajectories. Convex hull 
    backups can be viewed as the multi-objective analogue of 
    Bellman backups: rather than maintaining a scalar average 
    return at each node, they maintain a set of achievable 
    value vectors that more accurately represents the range 
    of outcomes reachable from that node. The CZT ball value 
    estimates $\Qczt(B)$ provide a coarser representation, 
    averaging over a region of weight space rather than 
    capturing the full structure of achievable values.

    Since the CHMCTS backups (Equations 
    (\ref{eq:5:chmcts_v_backup}) and 
    (\ref{eq:5:chmcts_q_backup})) are independent of the 
    search policy, they can be combined with any search 
    policy, including CZT. The resulting algorithm, 
    \textit{Convex Hull CZT} (CH-CZT), runs the CZT 
    algorithm but maintains additional convex hull value 
    sets at each node and uses the recommendation policy 
    from Equation (\ref{eq:5:chmcts_rec}) in place of the 
    CZT recommendation policy. This combines the contextual 
    exploration of CZT with the richer value representation 
    of CHMCTS.

\subsection{Optimisations}
\label{sec:5-6-optimisations}

    Two practical optimisations are used in the implementation of CHMCTS: \textit{solved labelling}, which avoids wasting computation on well-explored parts of the search tree; and \textit{size pruning}, which limits the number of points stored in each convex hull value set.

\paragraph{Solved Labelling.}

    \newcommand{\unsolved}{{\textnormal{\texttt{unsolved}}}}
    \newcommand{\unsolvedtol}{{\textnormal{\texttt{unsolved}}_{\textnormal{tol}}}}
    \newcommand{\deltaunsolved}{{\delta_{\textnormal{unsolved}}}}
    \newcommand{\sucobs}[2]{{\textnormal{Succ}_{\textnormal{obs}}(#1,#2)}}

    In planning, \textit{labelling} \todo{cite lrtdp} is used to mark nodes as solved and prune them from further consideration, avoiding wasted computation on parts of the tree that are already solved. A node is marked as solved when its value estimate has converged to the true value, \bd{which is enabled in the context of planning with full knowledge of the transition distribution}. In the setting considered here, the true transition distribution $p(\cdot|s,a)$ is unknown, so a probabilistic analogue for labelling is used instead.

    Each chance node $(s,a)$ is assigned an \textit{unsolved value} $\unsolved(s,a)\in[0,1]$, where $\unsolved(s,a)=0$ indicates that the node is solved and $\unsolved(s,a)=1$ indicates that it is unsolved. The solved value of a node is an upper bound on the probability that the node's value estimate has not yet converged.

    The unsolved value of a decision node is computed as the maximum unsolved value of its children:
    \begin{align}
        \unsolved(s) = \max_{a\in\cl{A}} \unsolved(s,a).
    \end{align}

    The unsolved values are updated during the backpropagation phase of \thtspp\ewe and used during the selection phase to filter out actions that have already been solved, to a tolerance of $\unsolvedtol$, with high probability. That is, at a decision node $s$, any action $a$ with $\unsolved(s,a) \leq \unsolvedtol$ is filtered out before selecting an action. The remainder of this section describes how the unsolved values are propagated through chance nodes, that is, how the value of $\unsolved(s,a)$ is computed.

    Ideally $\unsolved(s,a)$ would be computed either as an expectation or maximisation over the successor states. \bd{The issue with expectations is that the expectation using the empirical distribution may be overly optimistic (for example if there is only one child) about how solved the node is. The issue with maximisation is when there a successor state that is only sampled with low probability.} \bd{Some connecting hence like sentence}. \bd{Use an upper bound on the expected value instead}

    The solved value of a chance node $(s,a)$ is computed as an upper bound on the expected solved values of its successor states, using the empirical transition distribution $\frac{N(s')}{N(s,a)}$ as a proxy for the true distribution $p(s'|s,a)$. Since the true distribution is unknown, the bound must account for the estimation error in the empirical distribution. Let $\deltaunsolved > 0$ be a confidence parameter, \bd{and we will design the update for unsolved(s,a) such that}:
    \begin{align}
        \Pr\left(\bb{E}_{s'\sim p(\cdot|s,a)}[\unsolved(s')] \leq \unsolved(s,a)\right) \geq 1-\deltaunsolved. \label{eq:5:unsolved_chance_defn}
    \end{align}

    In words, with probability at least $1-\deltaunsolved$, the expected solved value of the children of chance node $(s,a)$ is bounded by the upper bound $\unsolved(s,a)$.

    \bd{Connecting sentence?}

    The expected solved value is decomposed into contributions from observed and unobserved successor states:
    \begin{align}
        \mathbb{E}[\unsolved(s')] 
            =& \sum_{s'\in\sucobs{s}{a}} 
                p(s'|s,a)\unsolved(s') \label{eq:5:local6} \\
            & + \sum_{s'\notin\sucobs{s}{a}} 
                p(s'|s,a)\unsolved(s'), \label{eq:5:local7}
    \end{align}
    where $\sucobs{s}{a}= \{s'\in\suc{s}{a} \mid N(s') > 0\}$ is the set of successor states observed so far.

    For the observed successors, Corollary \ref{cor:4:bound_transition_distribution} of the DKW inequality (Theorem \ref{thrm:2:dkw_inequality}) gives with probability at least $1-\frac{\deltaunsolved}{2}$ for all $s'\in\sucobs{s}{a}$:
    \begin{align}
        p(s'|s,a) \leq \frac{N(s')}{N(s,a)} + 
            2\sqrt{\frac{\log(4/\deltaunsolved)}{2N(s,a)}},
    \end{align}
    so the contribution of observed successors in (\ref{eq:5:local6}) is bounded by:
    \begin{align}
        &\sum_{s'\in\sucobs{s}{a}} 
            p(s'|s,a)\unsolved(s') \\
        \leq & \sum_{s'\in\sucobs{s}{a}} 
            \left(\frac{N(s')}{N(s,a)} + 
            2\sqrt{\frac{\log(4/\deltaunsolved)}{2N(s,a)}}\right)\unsolved(s').
    \end{align}

    The \textit{Good-Turing estimate} (Section \ref{sec:2-5-convergence}) for the missing mass is:
    \begin{align}
        \goodturingestimate(s,a) 
            = \sum_{s'\in\suc{s}{a}}\frac{\one\left[N(s')=1\right]}{N(s,a)}.
    \end{align}

    For the unobserved successors, since solved values lie in $[0,1]$ and unsolved nodes, the contribution of unobserved successors (\ref{eq:5:local7}) is bounded by the missing mass using Theorem \ref{thrm:2:good_turing_bound} with probability at least $1-\frac{\deltaunsolved}{2}$:
    \begin{align}
        \sum_{s'\in\sucobs{s}{a}} p(s'|s,a) 
        \leq \goodturingestimate(s,a) + \sqrt{\frac{\log(2/\deltaunsolved)}{N(s,a)}}.
    \end{align}
    
    Combining these bounds via the union bound, the following assignment of $\unsolved(s,a)$ satisfies (\ref{eq:5:unsolved_chance_defn}):
    \begin{align}
        \unsolved(s,a) \leftarrow &
            \sum_{s'\in\sucobs{s}{a}} 
            \left(\frac{N(s')}{N(s,a)} + 
            2\sqrt{\frac{\log(4/\deltaunsolved)}{2N(s,a)}}\right)\unsolved(s') \\
            + &\ \goodturingestimate(s,a) + \sqrt{\frac{\log(2/\deltaunsolved)}{N(s,a)}}.
            \label{eq:5:solved_value}
    \end{align}
    
    When the $\unsolved$ labelling is used in CHMCTS, $\unsolvedtol$ and $\deltaunsolved$ are hyperparameters of the algorithm. A node is marked as solved when $\unsolved(s,a) \leq \unsolvedtol$, at which point it is excluded from further trials.

\paragraph{Convex Hull Size Pruning.}

    As the search progresses, the convex hull value sets at each node may grow large, increasing the computational cost of the $\cprune$ operation and the convex hull backups. To limit this cost, a maximum size $K_{\max}$ is imposed on each convex hull value set. When a value set exceeds this limit after a backup, points are pruned using the \textit{crowding distance} from the NSGA-II algorithm \todo{cite}. % \cite{nsga2}.

    The crowding distance of a point $\bff{v}$ in a set $\bfcl{V}$ measures how isolated $\bff{v}$ is from its neighbours in objective space. For each objective dimension $d=1,\ldots,D$, the points in $\bfcl{V}$ are sorted by their $d$th coordinate. The boundary points (those with the smallest and largest values in dimension $d$) are assigned an infinite crowding distance and are always retained. For interior points, the crowding distance contribution from dimension $d$ is:
    \begin{align}
        \textnormal{cd}_d(\bff{v}) = 
            \frac{v^+_d - v^-_d}{\max_{\bff{v}'\in\bfcl{V}} 
            v'_d - \min_{\bff{v}'\in\bfcl{V}} v'_d},
    \end{align}
    where $v^+_d$ and $v^-_d$ are the $d$th coordinates of the nearest neighbours of $\bff{v}$ above and below in dimension $d$ respectively. The total crowding distance is $\textnormal{cd}(\bff{v}) = \sum_{d=1}^D \textnormal{cd}_d(\bff{v})$. When the value set exceeds $K_{\max}$ points, the $K_{\max}$ points with the largest crowding distances are retained, preserving the spread of the convex hull.

\subsection{Entropy and Convex Hulls}
\label{sec:5-7-entropy-convex-hulls}

    \begin{figure}
        \centering
        \includegraphics[width=0.6\textwidth]{figures/todo.jpg}
        \caption[Entropy and convex hull value sets.]{\bd{A 
        figure illustrating that a single point on the convex 
        hull may be optimal for many different weight vectors, 
        each corresponding to a different policy with a 
        different entropy value.} \todo{two convex hulls, one with two weights with same value, and a downstream state with different values for the same two weights}}
        \label{fig:5:entropy_convex_hull}
    \end{figure}

    Convex hull value sets represent the set of achievable expected returns in \textit{objective space}, storing value vectors attainable by different policies. Entropy, by contrast, is a property of a policy rather than a value vector: it measures the uncertainty of the action distribution induced by a policy at each state.

    A key observation is that a single point on the convex hull does not correspond to a unique policy, illustrated in Figure \ref{fig:5:entropy_convex_hull}. Since the contextual policies considered in this chapter depend on the weight vector $\bff{w}$, a given value vector $\bff{v}\in\Valg(s)$ may be optimal for many different weight vectors, each inducing a different policy $\pi(\cdot|s;\bff{w})$ with a different entropy. 

    As a result, there is not a principled way to associate a single entropy value with a point on the convex hull. This is the fundamental reason why the entropy-based exploration used in DENTS (Chapter \ref{ch:4-dents}) does not extend naturally to the multi-objective setting with convex hull value sets, and motivates the simplex map approach introduced in Chapter \ref{ch:6-simplexmaps}.

        selection rule reduces to standard single-objective action selection under 
        the scalarised reward $\bff{w} \cdot \bff{r}(s, a)$. Utility-optimality of 
        $\pi_{\mathrm{greedy}}$ then follows from the correctness of convex hull 
        value iteration (Theorem~\todo{ref CHVI}), which reduces to standard value 
        iteration for each fixed $\bff{w}$.
    
        \bd{Only if direction.}
        We prove the contrapositive by exhibiting a single MOMDP and utility 
        function for which $\pi_{\mathrm{greedy}}$ is not utility-optimal. Let $u$ 
        be the Chebyshev utility with reference point $\bff{z}^* = (6, 6)$,
        %
        \begin{equation*}
            u(\bff{v}; \bff{w}) = -\max_i w_i \left| v_i - z^*_i \right|,
        \end{equation*}
        %
        which is non-linear in its first argument. Consider the MOMDP in 
        Figure~\todo{ref fig}, which has a single action $a_0$ at $s_0$ 
        transitioning with equal probability to successor states $s_1$ or $s_2$, 
        at each of which the agent selects between $a_0$ and $a_1$ to receive a 
        terminal vector reward. The four deterministic paths yield
        %
        \begin{align*}
            s_0 \to s_1 \to a_0 &: (5, 0), &
            s_0 \to s_1 \to a_1 &: (2, 2), \\
            s_0 \to s_2 \to a_0 &: (2, 2), &
            s_0 \to s_2 \to a_1 &: (0, 5).
        \end{align*}
        %
        Fix the preference $\bff{w} = (\tfrac{1}{2}, \tfrac{1}{2})$. By inspection, 
        the four stationary policies yield expected values at $s_0$ of $(3.5, 1)$, 
        $(2.5, 2.5)$, $(2, 2)$, and $(1, 3.5)$, with the policy achieving 
        $(2.5, 2.5)$ being utility-optimal:
        %
        \begin{equation*}
            u((3.5, 1); \bff{w}) = -2.5, \quad
            u((2.5, 2.5); \bff{w}) = -1.75, \quad
            u((1, 3.5); \bff{w}) = -2.5.
        \end{equation*}
        %
        (The value $(2, 2)$ is Pareto-dominated and is omitted.) The utility-optimal 
        policy selects $a_0$ at $s_1$ (receiving $(5, 0)$) and $a_1$ at $s_2$ 
        (receiving $(0, 5)$), giving expected value $(2.5, 2.5)$ and utility $-1.75$.
    
        Now consider $\pi_{\mathrm{greedy}}$. The Pareto fronts at the successor 
        states are singletons,
        %
        \begin{align*}
            \bfcl{Q}^*(s_1, a_0) &= \{(5, 0)\}, &
            \bfcl{Q}^*(s_1, a_1) &= \{(2, 2)\}, \\
            \bfcl{Q}^*(s_2, a_0) &= \{(2, 2)\}, &
            \bfcl{Q}^*(s_2, a_1) &= \{(0, 5)\},
        \end{align*}
        %
        and the Chebyshev utilities are $u((5, 0); \bff{w}) = u((0, 5); \bff{w}) = -3$ 
        and $u((2, 2); \bff{w}) = -2$. Hence $\pi_{\mathrm{greedy}}$ selects $a_1$ at 
        $s_1$ and $a_0$ at $s_2$, both yielding $(2, 2)$. The expected value at $s_0$ is
        %
        \begin{equation*}
            \bff{V}^{\pi_{\mathrm{greedy}}}(s_0) 
            = \tfrac{1}{2}(2, 2) + \tfrac{1}{2}(2, 2) = (2, 2),
        \end{equation*}
        %
        with utility $u((2, 2); \bff{w}) = -2 < -1.75$. Therefore 
        $\pi_{\mathrm{greedy}}$ is not utility-optimal.
    
        The obstruction is structural: under Chebyshev utility the optimal policy 
        requires selecting the \emph{locally} worst continuation ($(5, 0)$ at $s_1$ 
        and $(0, 5)$ at $s_2$), whose expectation across the stochastic transition 
        yields a \emph{globally} preferable balanced outcome. No per-state $\argmax$ 
        on $u$ can discover this coordination, because $u$ does not commute with the 
        expectation over successor states. Linearity of $u$ is precisely the 
        condition under which this commutation holds, and hence precisely the 
        condition under which per-state action selection is utility-optimal.
    \end{proofoutline}

% Chebyshev scalarisation proof
% Assumes standard amsmath / amssymb / amsthm setup.
 
\begin{prop}
    Let $u(w, x, z) = \min_{i} w_i \lvert x_i - z_i \rvert$ denote the Chebyshev
    scalarisation, and consider the vectors
    \[
      a = (10, 0), \quad b = c = (1, 1), \quad d = (0, 10),
    \]
    together with a reference point $z \in \mathbb{R}^2$ satisfying $z_i < \min\{a_i, b_i, c_i, d_i\}$
    componentwise. Let $w = (w_1, w_2)$ be any weight vector with $w_1, w_2 > 0$ and
    $w_1 + w_2 = 1$. Then
    \[
      u(w, b, z) > u(w, a, z) \quad \text{or} \quad u(w, c, z) > u(w, d, z).
    \]
\end{prop}
     
    \begin{proof}
    The componentwise inequality $z_i < \min\{a_i, b_i, c_i, d_i\}$ permits the absolute
    values to be dropped, so that
    \[
      u(w, x, z) = \min_{i} w_i (x_i - z_i)
    \]
    for each of $x \in \{a, b, c, d\}$. The shifted coordinates
    \[
      \alpha_i = a_i - z_i, \qquad
      \beta_i  = b_i - z_i = c_i - z_i, \qquad
      \delta_i = d_i - z_i
    \]
    are therefore all strictly positive. From $z_1 < 0$ and $z_2 < 0$ (which follow from
    $z < d$ and $z < a$ respectively), the following strict inequalities are obtained:
    \begin{equation}\label{eq:key-ineqs}
      \alpha_2 = -z_2 < 1 - z_2 = \beta_2,
      \qquad
      \delta_1 = -z_1 < 1 - z_1 = \beta_1.
    \end{equation}
    Since $b = c$, the equality $u(w, b, z) = u(w, c, z)$ holds throughout.
     
    Suppose, for contradiction, that both disjuncts fail, i.e.
    \begin{align}
      \min(w_1 \alpha_1,\, w_2 \alpha_2) &\geq \min(w_1 \beta_1,\, w_2 \beta_2), \label{eq:A} \\
      \min(w_1 \delta_1,\, w_2 \delta_2) &\geq \min(w_1 \beta_1,\, w_2 \beta_2). \label{eq:D}
    \end{align}
     
    From \eqref{eq:A}, the left-hand side is bounded above by $w_2 \alpha_2$, so
    \[
      w_2 \alpha_2 \;\geq\; \min(w_1 \beta_1,\, w_2 \beta_2).
    \]
    By \eqref{eq:key-ineqs} we have $w_2 \alpha_2 < w_2 \beta_2$, whence the minimum on
    the right cannot equal $w_2 \beta_2$. It follows that
    \begin{equation}\label{eq:from-A}
      \min(w_1 \beta_1,\, w_2 \beta_2) = w_1 \beta_1, \qquad \text{so} \qquad w_1 \beta_1 \leq w_2 \beta_2.
    \end{equation}
     
    By the analogous argument applied to \eqref{eq:D}, using $w_1 \delta_1 < w_1 \beta_1$,
    \begin{equation}\label{eq:from-D}
      \min(w_1 \beta_1,\, w_2 \beta_2) = w_2 \beta_2, \qquad \text{so} \qquad w_2 \beta_2 \leq w_1 \beta_1.
    \end{equation}
     
    Combining \eqref{eq:from-A} and \eqref{eq:from-D} yields $w_1 \beta_1 = w_2 \beta_2$.
    Substituting back into \eqref{eq:A} gives $w_2 \alpha_2 \geq w_1 \beta_1 = w_2 \beta_2$,
    i.e.\ $\alpha_2 \geq \beta_2$, contradicting \eqref{eq:key-ineqs}.
     
    The assumption that both disjuncts fail is therefore untenable, and at least one of
    $u(w, b, z) > u(w, a, z)$ or $u(w, c, z) > u(w, d, z)$ must hold.
    \end{proof}
     
    \begin{remark}
    The argument relies only on the strict inequalities $a_2 < b_2$ and $d_1 < c_1$
    (together with $b = c$). Consequently the proposition generalises: whenever a common
    point $b = c$ strictly dominates $a$ in one coordinate and $d$ in the other, no
    single Chebyshev weight vector can simultaneously render both $a$ preferable to $b$
    and $d$ preferable to $c$.
    \end{remark}
